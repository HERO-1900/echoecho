## 一、手残党男生
> 我手指头粗、点不准、最怕误触和复杂手势。

1. **开屏页：创建/加入按钮**  
   - 毛病：昵称、小家名输入框只有一条细线，点击区域小，容易点歪；创建按钮在键盘上方飘来飘去。  
   - 建议：把输入框高度设为 **56px**，底部"创建小家"按钮做成 **fixed 底部 44px 高** 的大按钮；加入邀请码改成 6 个独立大方格（每个 48px），自动跳到下一格。

2. **主界面：顶部邀请码 chip**  
   - 毛病：小家名右侧的邀请码 chip 太小，想复制老点成设置。  
   - 建议：邀请码 chip 高度 **≥44px**，右侧留出 **8px 安全间距**，点击后 0.5s 内弹出 Toast"邀请码已复制"。

3. **任务卡：三态按钮**  
   - 毛病："收到→在办→办妥" 一串小按钮挨着，我一点就误标成"办妥"。  
   - 建议：单张卡只保留一个主状态按钮，尺寸 **≥48×48px**；"在办"切到"办妥"时弹底部确认面板"确定完成了吗？"，二次确认再变更。

4. **在办进度滑条**  
   - 毛病：进度滑条太细，我根本拖不准。  
   - 建议：滑条高度提到 **12px**，两边加 **"－ / ＋"按钮**，同时支持直接点击进度条跳转到目标百分比。

5. **底部输入条：语音按钮**  
   - 毛病：🎙️ 图标小，点开始和结束容易点错。  
   - 建议：麦克风按钮放大到 **56px**，点击开始后按钮变成红色"结束"大按钮，文字显示"再点一次结束录音"。

6. **方案页：底部三个小按钮**  
   - 毛病："回到主界面 / 分享 / 回响给TA" 三个底部按钮太挤，点错就退出了。  
   - 建议：底部操作栏高度 **56px**，按钮图标+文字，主按钮"回响给TA"用实心高亮，"回到主界面"放到左侧次要位置。

## 二、细节控女生
> 我对对齐、文案、反馈、状态一致性很敏感。

1. **开屏页：输入框占位符与 Loading**  
   - 毛病：输入框提示只有"昵称 / 小家名"，没有示例；创建按钮被连续点击会重复创建。  
   - 建议：占位符改成"例如：阿杰"、"例如：我们的小家"；创建按钮点击后显示 **Loading 状态并禁用**，防止重复提交。

2. **主界面：三张统计卡**  
   - 毛病：卡内数字、标签字号不统一，单位位置忽上忽下，空值时显示空白。  
   - 建议：数字统一 **24px 加粗**，标签统一 **14px** 置于数字下方，无数据时显示 **0**，三卡外边框和内间距保持 **16px** 一致。

3. **Tabs：全部 / 我发出的 / 我收到的**  
   - 毛病：没有未读数量提示，我不知道哪类需要处理。  
   - 建议：每个 Tab 右侧加 **红色数字角标**，显示未读/待处理数量；当前 Tab 下划线颜色与品牌色一致。

4. **搜索框**  
   - 毛病：搜索后没有清空按钮，输入框右侧空着。  
   - 建议：输入框右侧常驻 **"×"清空图标**，搜索结果中高亮匹配关键词。

5. **任务卡：时间戳与语音按钮**  
   - 毛病：时间格式有时"10分钟前"有时"2024-05-20"，很乱；"听原声"像文字链接，容易忽略。  
   - 建议：时间统一按"今天 09:30 / 昨天 / 周一"规则；"听原声"改成 **🔊 图标 + "听原声"** 的按钮样式。

6. **三态按钮文案与颜色**  
   - 毛病："收到 / 在办 / 办妥"像黑话，三个状态颜色都一样。  
   - 建议：状态色区分：**收到=蓝、在办=橙、办妥=绿**；文案改为"已收到 / 办理中 / 已完成"。

7. **AI 方案按钮状态变化**  
   - 毛病："请AI僚机出方案"点完没有 Loading，突然变成链接，吓人一跳。  
   - 建议：点击后按钮变 **"AI 正在想…" 进度状态**；方案生成后变成 **"查看 AI 方案" 按钮**，并在右侧加小圆点提示未读。

8. **设置页：成员列表**  
   - 毛病：头像大小不一、昵称和状态没有对齐。  
   - 建议：头像固定 **40px 圆角**，昵称 **16px 中等字重**，在线状态用绿色小圆点统一放在头像右下角。

## 三、50岁阿姨
> 我眼睛花、不爱看小字，技术词听不懂，怕点错。

1. **开屏页：无手机号、token 存在本地**  
   - 毛病：没有手机号，换个手机或清缓存，我的小家是不是没了？  
   - 建议：创建小家后顶部出现 **温馨提示条"数据已保存在本设备，建议绑定手机号或记下邀请码"**，并给出"复制邀请码"按钮。

2. **术语看不懂**  
   - 毛病："小家"、"回响"、"AI 僚机"都是年轻人的词，我搞不懂。  
   - 建议：首次出现这些词时，旁边显示 **黄色问号图标**，点击弹出白话解释，如"回响 = 对方看到并回复你"。

3. **主界面：统计卡字体和对比度**  
   - 毛病：数字小、颜色浅，我戴上老花镜也看不清。  
   - 建议：正文最小 **16px**，数字最小 **20px**，文字与背景对比度符合 WCAG AA（建议深灰 #333 配白底，避免浅灰）。

4. **Tabs：文案**  
   - 毛病："全部 / 我发出的 / 我收到的"没有图标，我看半天分不清。  
   - 建议：改成 **"全部任务 / 我发的 / 我收的"**，每个 Tab 加图标，比如"发出去"用向上箭头，"收到的"用向下箭头。

5. **任务卡：三态按钮**  
   - 毛病：我不知道按钮按下去会发生什么，害怕按错。  
   - 建议：按钮上方加一句小字提示当前该做什么，例如"收到任务后点这里"；点击后按钮变灰禁用，显示"已记录"。

6. **语音发送**  
   - 毛病：麦克风按钮小、录音状态不明显。  
   - 建议：录音按钮放大并变成 **红色大圆按钮**，录音中屏幕中间显示"正在录音，请说话，再点一下结束"。

7. **设置页：硬件 Token / 退出**  
   - 毛病："生成硬件设备 Token"、"退出"像电脑术语，我不敢点。  
   - 建议："生成硬件设备 Token" 改成 **"给智能音箱/手表用"**，"退出"改成 **"退出当前小家"**；"退出"按钮放在页面最底部，点击前弹确认框"确定退出吗？退出后需重新输入邀请码"。

8. **完成时：发起方通知**  
   - 毛病：我办了事，对方知不知道？  
   - 建议：我点"办妥"后，给我弹一个提示"已通知对方"，同时对方那边也要有显眼的提示。

## 四、程序员
> 我关注安全、稳定性、性能和可维护性。

1. **开屏：本地 token 无账号体系**  
   - 毛病：token 直接丢 localStorage，没过期、没刷新、换设备就丢。  
   - 建议：引入 **access_token + refresh_token**；access_token 用内存/HttpOnly Cookie，refresh_token 安全存储；创建/加入时引导用户绑定手机号作为账号恢复依据。

2. **主界面：12 秒轮询**  
   - 毛病：12 秒轮询既不及时又浪费资源，频繁重绘列表。  
   - 建议：优先改用 **WebSocket / SSE 长连接**；不具备条件时改用 **5 秒轮询 + 窗口获得焦点时立即刷新**。

3. **任务卡：AI 方案是完整 HTML**  
   - 毛病：直接渲染完整 HTML 有 XSS 风险，可能被 prompt injection 注入脚本。  
   - 建议：AI 输出改为 **Markdown**，前端用可信库渲染；必须保留 HTML 时先经 **DOMPurify 消毒**，再在 iframe sandbox 中展示，禁止 script。

4. **三态按钮：并发与状态回滚**  
   - 毛病：连续点击或网络抖动时，状态可能错乱或重复提交。  
   - 建议：按钮点击后 **立即 disabled + 显示 loading**，前端乐观更新，后端返回失败时回滚并提示"网络异常，请重试"。

5. **发送语音：转写过程不可见**  
   - 毛病：录音→上传→转写→发出，任一环节失败用户都感知不到。  
   - 建议：在输入条上方显示 **"上传中 / 转写中 / 发送中" 进度条**；失败时给出"重转文字"或"直接发送语音"选项。

6. **设置：硬件设备 Token 无权限管理**  
   - 毛病：生成的硬件 token 没有过期、没有范围、没有撤销入口。  
   - 建议：token 列表展示 **设备名称、生成时间、权限范围、过期时间**，支持"撤销"按钮，默认展示 masked 值。

7. **退出：未清理状态**  
   - 毛病：退出可能只是跳转，没有真正清除本地 token 和状态。  
   - 建议：退出时弹确认框，确认后 **调用 logout 接口并清除 localStorage / sessionStorage**，再跳转回开屏页。

8. **邀请码：无安全限制**  
   - 毛病：邀请码固定且可能被爆破。  
   - 建议：邀请码支持 **过期时间** 和 **最大使用次数**，新成员加入时可选"需要房主确认"。

## 五、异地恋
> 我在异地，最在意的是情感连接、时差和有没有被对方看到。

1. **开屏：小家名默认太冷冰冰**  
   - 毛病：默认小家名可能就是"小家"，没有情感。  
   - 建议：创建时默认填写 **"我们的小家"** 或给几个可选文案如"距离 1280km 的我们"，让用户感到亲近。

2. **任务卡：时间显示没有时区**  
   - 毛病：我早上 8 点发的任务，对方显示"3 小时前"，不知道各自几点。  
   - 建议：时间旁显示 **对方当地时间**（如"TA 那边 晚上 11 点"），或显示相对时间+当地时间 tooltip。

3. **在办进度：缺少情感同步**  
   - 毛病：只有"办妥"才通知，我在做任务的过程对方完全不知道。  
   - 建议："在办"进度条每次拖动后，可选 **"同步进度给 TA"**，发送一条轻量提醒"我正在做 35% 啦"。

4. **办妥：庆祝动效只有一方看到**  
   - 毛病：异地恋最想要"一起庆祝"的感觉，但发起方只能收到一个通知。  
   - 建议：执行方点"办妥"后，**双方都弹出全屏庆祝动画**（如撒花），并给对方发送手机推送+震动。

5. **方案页："回响给TA" 文案模糊**  
   - 毛病：我不知道这个按钮会帮我生成什么，会不会直接发错。  
   - 建议：按钮改为 **"生成暖心回复发给 TA"**，点击前弹窗预览回复内容，确认后再发送。

6. **语音任务：对方没听原声**  
   - 毛病：转文字后，对方可能以为看完文字就行，忽略了语气。  
   - 建议：未听原声的语音卡片显示 **红色小圆点**，播放时显示进度条，文字下方提示"听听 TA 的原声"。

7. **主界面：统计卡太功利**  
   - 毛病：全是办妥率、方案数，没有情感维度。  
   - 建议：加一张 **"我们的里程碑"** 卡片，显示连续完成任务天数、累计回响次数。

8. **搜索：翻旧承诺**  
   - 毛病：异地恋会翻旧账找承诺，但搜索只能按文字。  
   - 建议：搜索结果支持按 **时间、状态（已办/待办）、发起人** 筛选。

## 六、新用户第一次打开
> 我刚打开，完全不知道这是干嘛的，别让我猜。

1. **开屏：没有引导**  
   - 毛病：一进来就让填"小家名"和"昵称"，我不知道要做什么。  
   - 建议：增加 **3 步轮播引导**：1. 创建只属于你们的小家；2. 给 TA 发一件小事；3. 用邀请码让 TA 加入。轮播底部给出"创建小家"和"输入邀请码"两个大按钮。

2. **创建成功后：没有立即邀请对方**  
   - 毛病：创建完进了主界面，我不知道怎么把对方拉进来。  
   - 建议：创建成功后立刻弹出 **邀请码大卡片**（邀请码 28px 加粗显示），并提供 **"复制邀请码，去微信发给 TA"** 按钮。

3. **主界面：空状态没有示例**  
   - 毛病：列表空白，我完全不知道该怎么发第一条。  
   - 建议：空状态时显示 **一张示例任务卡**（半透明或"示例"标签），文案如"给 TA 发一句：记得喝牛奶"，下方提示"点底部输入框开始"。

4. **底部输入条：没有首次提示**  
   - 毛病：输入条不显眼，新手不知道在哪发任务。  
   - 建议：首次进入主界面时，输入条外侧出现 **黄色描边 + 浮层提示"从这里给 TA 发任务"**。

5. **AI 僚机：用途不明**  
   - 毛病：卡片上"请AI僚机出方案"不知道有什么用。  
   - 建议：首次显示 AI 按钮时，弹出 **教练气泡"不知道怎么做？让 AI 帮你想方案"**。

6. **三态按钮：缺少新手说明**  
   - 毛病：收到任务后我不知道要点按钮。  
   - 建议：第一条任务卡上的"收到"按钮旁显示 **脉冲高亮动画 + 小字"点我确认收到"**。

## 八、新手首次体验的 3 个流失点

1. **开屏不知道"小家"是什么，且担心没有账号会丢数据**  
   - **流失表现**：看到"昵称 + 小家名"就关掉，或不敢创建。  
   - **修改建议**：增加 **3 步引导轮播**；创建完成后立即显示 **"数据已保存在本设备，建议复制邀请码或绑定手机号"** 的提示条，降低心理不安全感。

2. **进入主界面后一片空白，没有示例和明确入口**  
   - **流失表现**：新手不知道底部输入条是做什么的，逛两秒就离开。  
   - **修改建议**：空状态展示 **示例任务卡**（半透明，带"示例"标签）；首次进入时 **高亮底部输入条**并浮层提示"给 TA 发一件小事"。

3. **第一条任务发出后，三态按钮和 AI 按钮无引导，用户不知如何完成闭环**  
   - **流失表现**：发出任务后没有反馈，也不知道"收到→在办→办妥"的用处，AI 按钮也不明白。  
   - **修改建议**：第一条任务卡上的"收到"按钮做 **脉冲高亮 + 小字"点我确认收到"**；首次出现 AI 按钮时显示 **教练气泡"点这里，AI 帮你出方案"**；状态变更后给明确 Toast 反馈。
