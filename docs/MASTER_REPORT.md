# 1. 执行摘要

## 1.1 一句话是什么

**念念 EchoEcho,是一条"说一句,就有着落"的最短的路。**

她随口说一句话(语音或文字)→ 在他那边自动变成一张任务卡 → 收到 / 在办 / 办妥三态流转 → 她收到回响。AI 僚机能把一句抱怨自动翻译成一页可执行方案。它不是备忘录,是**关系里的一条"承诺协议"(promise protocol)**——把一句话从"说出口"到"有着落"的全链路(捕获→理解→分配→执行→回执→记忆)兜住,让善意的、在场的、只是被生活噪音冲散了的爱,不再漏接。

对外的核心一句话:**AI 把"记住"的成本,转移给机器;把"被记住"的感受,留给人。**(`AI_NATIVE_VALUE.md` §4 候选 A)

## 1.3 核心结论(七条)

1. **这个品类的及格线,今天才第一次被够到。** 2012 年的 Avocado 做的就是"没有 AI 的念念",团队来自 Google,照样死了——它输给的不是需求或想法,是"把她的话变成结构化任务、把任务变成可执行方案"的那个**劳动力**当年不存在。LLM 第一次同时压掉了录入成本(C1)、执行成本(C3)并中介化了协调成本(C2)。(`AI_NATIVE_VALUE.md` §2/§3)
2. **别人卖甜,我们卖"作数"。** 恋爱记 3000 万注册仍靠游戏化道具找收入——情绪共鸣带来下载,带不来付费。付费假设必须钉死在"着落"上,不能钉在"甜"上,因为"忘了→吵架"这条链路有明确的失败成本、清晰的责任方与购买方、且高频可验证。(`MARKET.md`)
3. **三条一票否决红线是全部克制的来源:不做监工、不做证据本、不做 KPI。** 两份 persona 独立收敛出同一结论:执行方最高频的卸载理由不是功能少,而是"被监控、被考核、被翻旧账"。这三条在日本(管理焦虑)、韩国(性别断层线)被独立地再验证了一遍。
4. **第一版可能靠"冲突感"而非"甜蜜"破圈。** 社媒 + 舰队高度收敛:甜蜜是消费品,毒舌是创作素材;抖音 / 小红书 99% 情侣内容都在卷"他好宠我",用户审美疲劳。"小狗 / 修勾"是自带 meme 属性、传播量最高的引擎,但它是流量不是地基。地基看"念想",流量看"小狗",蓝海看"爸妈",硬件付费意愿看"异地恋"。
5. **首发日本,不是文化最贴合的韩国。** 文化最贴合的是韩国(拉勾天生带"拇指盖章"、Between 刚制造信任真空);但**日本刚刚空出来**——本土情侣 App 全线阵亡,当前 App Store 前三名全是韩国货,唯一本土 AI 情侣产品上线一年仅约 4000 对情侣。加上日本 2025-12 智能手机新法可合法绕开 30% IAP 抽成,这是一个有时效的窗口。
6. **拉勾仪式是中日韩美同源的最大出海资产,但它是"内核 + 视觉",不是"名字"。** 从日本江户吉原的「指切り」向外扩散的同一个仪式,四国的这个词各自都被寄生或占用了(拉勾网、货拉拉、Pinky Swear 儿童癌症基金会、日本 YUBIKIRI 已注册且命中交友类目)。可全球统一的是手势视觉(两拳小指相扣、负空间藏心、缩小成 ✓)与产品语法(勾=立约、章=兑现);名字必须四国四份。
7. **技术路线明确:iOS 原生 SwiftUI + 复用现有 Cloudflare 后端。** 灵动岛 / Live Activity / 锁屏组件在 RN / Flutter / Capacitor 里都要回到 SwiftUI 写 Widget Extension,而"跨端"唯一收益(iOS+Android 共享)因安卓暂缓而归零。后端零迁移,只需补 APNs 推送与真账号体系。

# 2. 产品定义(调研后的最终版)

## 2.1 核心范式:说一句,就有着落

产品的骨架一句话说清:**她说一句(语音或文字)→ 他那边自动变成一张任务卡 → 收到 / 在办(带进度%)/ 办妥(可附一句交待)→ 她收到回响。** AI 僚机能把一句话自动变成一页可执行方案。就这么简单——简单不是能力不足,是刻意。最好的工具是感觉不到它存在的工具(海德格尔的"上手状态")。她按住说一句话,意识里不该有"我在操作一个 App",只该有"我在跟他说件事"。

三态不是随便设计的进度条,它精确对应一句社会金句——"凡事有交待、事事有回应、件件有着落":

- **收到 = 凡事有交待。** 最轻、也最关键的一态。它是回应性理论里那个"我在,我看到了"的最低成本信号,是对"沉默"这个最伤人东西的第一道防线。物理离线的场景(军营、手术室、开会),"收到"存在的全部意义是当"消失感"的解毒剂——让"看不到"不再自动翻译成"不在乎"。
- **在办(带进度%)= 事事有回应。** 它把"等待"制度化——等待不再是"他为什么还不回",而是"这件事在被认真对待的路上"。进度%的语言包装永远是"经验条 / 着落感",**永远不做"完成率 / 效率 / 成绩单"**(一票否决红线)。它把"催"变成了"看"。
- **办妥(可附一句交待)= 件件有着落。** 整个产品的情感高潮。所有待办 App 的完成态都是冷冰冰的"已完成",只有这个,是**替对方卸下心里的那份重量**。它触发的"回响仪式"——一句温暖的、绝不用系统腔的通知("你两周前提的床头灯,TA 办妥了")——是接收方(尤其是那些婚姻里稀缺了十年正反馈的男人)的 Aha 时刻。

这三态之所以是"正确的产品形态",因为它同时做对了三件事:承载了点对点的方向性(谁欠谁)、用"收到"消灭了沉默、用"办妥"完成了从承诺到兑现的仪式闭环。缺任何一态,协议就不成立。

## 2.2 五模式定位(五份用户研究校准后的最终版)

70% 的底盘(数据模型 / API / 交互流程)完全一致,模式 =「视觉 token + 话术包 + 少量特调开关」,存 localStorage(设备级人格,双方可各用各的模式)。五种人格对应关系光谱上的不同位置。以下是**结合五份 6-persona 用户研究校准后的定位**,而非 MODE_SPEC 的初版设定。

### A 念想 / 念响(默认 · 成熟同居的"过日子")

- **人群**:同居 / 婚姻 / 半同居的成熟稳定关系,28–40 岁,过日子的生活琐事协同刚需——物业费、换季收纳、双方父母的事、囤货、维修工上门。
- **核心校准(6 个 persona 独立收敛)**:去掉情侣撒娇腔。"办妥 ✓ 你做到了""🌸 有回响"用在物业费、报修、囤货这些行政 / 家务事务上是**错位**的——不是不需要温暖,是温暖的形态要从"夸奖式甜蜜"改成"接住式踏实"。文案应改成"办好了,家里我担着"这类**平视的队友口吻**,而不是哄小孩式的"你做到了"。
- **三个第一等公民功能**:① 重复性事务模板化(物业费 / 续费 / 体检 / 疫苗设成"周期念想",这是比纪念日更冷更硬的品类,须独立呈现而非藏在纪念日功能里);② 双方父母事项单独建模,且分两种路径——"行政协调型"(档期撞车、顺路探望没说清)给日历式工具,"价值判断型"(催生、以什么身份探病)**不应被拆成任务卡或让 AI 出方案**,产品只做"提醒你们该聊聊了";③ 统计层从"办妥率"改成"这周我们一起办成的事"(6 人里 3 人独立警惕百分比会被拿来比较、放大出差方结构性劣势、被家人看到引发联想)。
- **本质区别**:通用 to-do 管理"任务",念想管理"被回应感";通用 to-do 对逾期是惩罚性 UI,念想该是温和补救;通用 to-do 的终局是清空列表,念想的终局是关系本身被照顾好。

### B 拉勾勾(恋爱前中期的"甜蜜仪式")

- **人群**:校园情侣、刚在一起、异地大学生、暧昧转正、第一次谈恋爱、网恋奔现——仪式感需求最强的阶段。
- **核心校准**:6 个 persona 几乎都提到盖章那一刻最想做的是"截图存进回忆相册",而不是拿去比分数。所以**盖章要"存证化"而非"计分化"**;"兑现率"这种百分比容易被过度解读成 KPI / 考试分数(在暧昧转正期和第一次恋爱者身上杀伤力最大),默认应弱化,改为"我们已经拉勾 N 次""连续 X 周说到做到"这种**加法叙事**而非分子分母的"率"。
- **四条反催命设计原则**(从 pinky.md 收敛):① 永远不做"未完成计时"式的施压视觉(D 模式的"已鸽 N 天"角标绝不挪到 B 模式);② 数字永远做加法叙事不做比率叙事;③ 允许"不拉勾"的沉默选项,不强制每句话走完整三态;④ 办妥后的社交暴露可控(默认通知折叠内容,仅显示"有新消息",避免办公室 / 同事场景社死)。
- **能力可调**:暧昧转正期需要"话术强度渐进开关"(从轻量确认过渡到正式拉钩),异地场景需要"倒数提前量可调"(提前一周预警而非当天),网恋 / 暧昧需要"多纪念日并存"。

### C 女王陛下(权力反差的"双向玩梗")—— 双支线是关键校准

- **人群**:自愿玩权力反差的情侣 / 个体。**关键升级:这不是单向"女对男"的性别刻板印象,而是权力反差本身可以双向流动**——女王 / 忠犬、皇上 / 爱妃、S / M,谁在上位、谁被照顾,由这对情侣自己定,不由性别预设。
- **最核心校准(Persona 2、3 共同诉求)——必须拆出双支线**:默认话术包不能只有"女王使唤忠犬"一种质地。至少拆出两条——**「命令-服从」**(女王 / 爱卿、皇上 / 臣子,执行方是"能干但需要指挥的下属")和**「恩宠-撒娇」**(皇上宠爱妃式,执行方不是被使唤的"狗",而是**主动去疼人**的一方,完成后是被夸奖赏赐,而不是"求摸头"这种讨好姿态倒挂的词)。谁拿"上位头衔"、用哪条话术支线,由情侣自己选,绝不预设"发号施令的就该被当狗对待"。
- **安全阀是一等公民**(Persona 1、4、5、6 反复强调):① 双人握手确认取代单方警告(开启带权力话术的模式,双方各自在自己设备上主动确认一次,类似"邀请-接受"流程,不能让一方毫无准备就收到"爱妃有新旨意");② 随时可退、零阻力、不问原因(退出机制必须比进入机制更轻,一句安全词或常驻按钮秒退,**绝不允许"确定要退出吗"式二次挽留——那不是安全阀,是给退出添堵**);③ 敏感内容强制脱离角色皮肤(医疗、金钱、人身安全类关键词一律中性语言呈现,优先级高于任何用户自定义,不能被"这周你俩谈好了可以更严格"绕过);④ 内置"红黄绿灯"信号系统(黄灯=放慢 / 确认的中间态);⑤ 视觉与话术分强度档位(经典宫廷 / 网络热梗 / 低调温润三档,适配老夫老妻的收敛需求与亚文化圈层的"scene 开场-结束"完整流程)。
- **不物化**:系统与 AI 军师生成的所有文本只围绕"任务是否办妥",绝不生成涉及外貌、身材、性化措辞的内容——这既是保护,也是这套玩法能被公开谈论、经得起审视的前提。

### D 毒舌修勾(小狗情结的"互怼玩梗")—— 毒舌是壳,小狗是核

- **人群**:年轻到中年情侣(22–39 岁),用损来表达亲密。**创始人新口径:D 模式真实使用者是情侣,核心不是"谁监管谁",而是普遍存在的"小狗情结"**——情侣之间(无论男女)互相希望对方像小狗一样听话、粘人、忠诚。
- **最核心校准(roastpup.md 全员共识)**:**毒舌是调味,不是主菜;小狗陪伴感才是那个"最忠诚的陪伴"的核心承诺。** 6 个 persona 没有一个人认为这两个调性互斥,也没有一个人认为该五五开——共识是"分工不是竞争":毒舌互怼是外壳,小狗陪伴感是内核。毒舌的真实功能是**"给情话一个不尴尬的出口"**——直接说"我爱你"在很多关系阶段说不出口,套上"汪汪我在""狗狗汇报 11 年没换主人"这层玩梗外壳,肉麻话就有了台阶下。
- **五条产品建议**:① 摇尾巴反馈不只挂在"办妥"上(任务没做完也要有"抱抱"级低强度正反馈,否则滑向"只有交作业才配得到爱");② 忠诚度做成"陪伴时间轴"不做排行榜(往"在一起 N 天没换主人"这种纪念感设计);③ 小狗日记自动生成(把任务流水转写成第一人称小狗视角周记,是"对抗式互动"沉淀成"深层连结"的关键转化器);④ 毒舌烈度可调 + 自动降火(按年龄 / 关系阶段分档,监测短时间内"通缉令"频次异常触发"你们是不是真的在吵架?"冷静期);⑤ 对抗式内容默认双人可见、不外显分享(社死的真实触发点是"当众",不是话术本身太狠)。
- **每一次"怼"必须有一次"和好"收尾**(Persona 3 明确提出的强制仪式,与 Gottman 婚姻研究"正负互动 5:1"逻辑一致):负反馈符号(通缉令、阴阳怪气值)必须有对应正反馈符号(和好 / 摸头 / 赏饼干)兜底,且量级要够。

### E 爸妈听话 / 牵挂(长辈关怀)—— 主 App 模式非独立 App,但要重做

- **人群**:异地照护父母的成年子女(付费方是焦虑的子女)+ 父母。E 模式核心不是"子女管教父母",而是**双向牵挂的翻转**——小时候爸妈教我们"听话",现在换我们啰嗦爸妈"听话"(按时吃药、复诊、别硬扛),同时父母也有自己的念叨(催婚、催回家、催添衣)。
- **最核心校准——四条界限(parents.md,几乎每位 persona 精确指出关心滑向监控的临界点)**:① 健康类提醒与"念叨 / 催婚"类话题**必须分渠道展示**,不共用同一张大字卡的视觉权重(高秀云案例:一旦"记得吃早饭"和"你张阿姨女儿不错哦"共用同套"牵挂"外壳,接收方会对整个系统的所有消息产生"下一句是不是又要催"的防御性预判,连累健康提醒的可信度);② "安静侦测"必须锁定在"提醒有没有被看到"这一件事,**不能演化成生命体征仪表盘**(周建国:术后已因疾病丧失自主感,做成实时血压心率监护面板会比生病本身更沮丧;触发阈值保持在 24 小时"异常"量级,不做实时化、不做数值化——对应"三不一只":不看位置、不听通话、不翻相册,只看提醒有没有被看到);③ 大字化不能只停留在字号,**对完全不会用智能机的老人 App 本身可能是无效触达**(林秀兰:通知栏、下滑查看、按住说话本身就是学习门槛;GTM 路线图里"床头一键呼叫盒""冰箱贴提醒屏"这类零学习成本物理入口,建议至少一个纳入 MVP 验证);④ 话术委婉度要按"关系身份"分层(黄敏婆媳案例:同一句"记得吃药",对亲妈是孝顺,对婆婆可能被解读为"媳妇越界管教")。
- **产品归属结论(舰队与用户研究双重反对硬塞)**:爸妈模式**不应作为情侣 App 的第五个模式硬塞**——对年轻情侣画风断裂,对父母"知道自己只是一个模式、不是主角"尊严感不够。**正确做法:共内核(结对 / 三态 / 语音 / 硬件 API 全复用)、不共界面入口**——老人打开的应该是一个只有三个大按钮的界面,而不是带模式切换器的情侣 App。作为"家人版"独立调性,而非"爸妈听话"这个略带冒犯的名字(老周:"我养你三十年,现在倒过来要我听你的话?")。

## 2.3 五模式如何用 70% 共用底盘承载(话术包 + 视觉 token)

五模式不是五个产品,是同一个内核换皮肤——这是它能低成本存在的工程前提。70% 的底盘(数据模型 / API / 交互流程)完全一致,模式 =「视觉 token 组 + 话术包 + 少量特调开关」,切换入口是右上角「模式」chip + 设置面板,存 localStorage(设备级人格,双方可各用各的模式,对方看到的是自己模式下的话术)。

**话术包差异**举例(同一个"一条消息"在五模式里的叫法):念响=念念、拉勾勾=勾勾、女王=圣旨、修勾=任务、爸妈=牵挂;同一个 received 态:念响=收到、拉勾=拉勾成立、女王=汪!遵命、修勾=汪,别催、爸妈=看到了;同一个 done 态:念响=办妥、拉勾=盖章兑现、女王=办妥·求摸头、修勾=交作业!、爸妈=办好了。**视觉 token 差异**(CSS 变量组):念响暖纸底 + 珊瑚 + 衬线字 + 涟漪波纹;拉勾奶油色 + 朱砂 + 圆体 + 契约虚线边 + 印章;女王曜石黑 + 鎏金 + 深卡金细边 + 火漆圣旨;修勾米绿 + 荧光绿 + 粗黑描边 + 便利贴胶带条 + 微旋转;爸妈米黄 + 枣红 + 大号 sans + 极简无装饰。**特调开关**:E 适老开关(字号×1.25、统计卡收一行、进度条隐藏百分比改文字"在办呢")、C 安全阀(切入弹一次确认,应升级为双人握手)、D 通缉令(超 48h 未动加"已鸽 N 天"角标,玩梗不发对方通知)。

这套多人格矩阵在原生 iOS 里的实现代价很低:`MODE_SPEC.md` 已是 CSS 变量组 + 话术总表,直译成 SwiftUI Design Token(`@Environment` + 枚举)即可秒切,70% 底盘不动。它的战略意义是:同一个"说一句→有着落"的协议,可以既服务 20 岁热恋的学生、又服务 38 岁过日子的夫妻——皮肤是低成本的 A/B 试验田,哪个模式数据跑赢就升格为独立系列去做硬件。但要清醒:根据校准结论,五模式绝不是"平均用力铺满",而是有明确分工(念想留存 / 拉勾获客 / 修勾破圈 / 女王传播 / 家人蓝海),且爸妈 / 家人必须物理隔离界面入口。

## 2.4 底层是六环"承诺协议",不是记录工具

把对象钉死,否则后面全是空转:念念的本质不是记录工具,是关系里的一条"承诺协议(promise protocol)"——一句话从"说出口"到"有着落"的全链路,有六个环节:捕获(她说)→ 理解(变任务)→ 分配(到他)→ 执行(他办)→ 回执(办妥回响)→ 记忆(沉淀为关系资产)。每一环都有真实成本,历史上所有同类产品的死法归根到底都是"某几环的成本高于用户感知的价值"。前 LLM 时代的传话工具要求用户支付三重成本:C1 录入成本(把话变成结构化条目的文书劳动)、C2 协调成本("给对象派任务"的社交成本,在待办软件里给男朋友建任务语感是上级派活,这在亲密关系里是有价签的)、C3 执行成本(办事本身)。前代工具只提供"存储",三重成本一个都没降,于是理性的人会算这笔账:在 App 里郑重其事建一条任务,还不如微信随口一句加人脑硬记来得便宜,工具死于"用它比不用它更累"。LLM 第一次同时压掉两个半——C1 趋近零(说话即结构化,录入从"新增劳动"变成"截获既有的抱怨")、C3 减半(AI 僚机自动出方案,他只剩确认与执行)、C2 被中介消解(任务卡不是她建的,是 AI 从她的话里长出来的,"她派活给他"变成"AI 替他整理了她的心愿",指令经过第三方转译就不再是指令)。理解了这六环和三重成本,就理解了为什么这个品类第一次出现"价值 > 成本"的交点,以及为什么这是一个"AI 时代才成立"的产品。

## 2.5 五模式是一条情感光谱,不是并列菜单

社会聆听的关键洞察(`SOCIAL_LISTENING.md` §3):五个模式不是并列,而是一条**情感光谱**,可由同一产品的不同皮肤承接——念想(工具刚需 / 信任地基,最稳但楔子最难)→ 拉勾(承诺凭证 / 仪式)→ 小狗 · 女王(情绪价值 / 传播引擎,最猛)→ 爸妈听话(代际外溢 / 最空的蓝海)→ 异地恋轻信号(硬件付费意愿最强)。这套多人格矩阵学的是泡泡玛特——同一套供应链、渠道、会员体系,上面跑 N 个"情绪人格";IP 不按年龄分,按"用户想借它表达哪种自我"分。它同时解决一个产品难题:同一个底层协议,如何既服务 20 岁热恋的学生,又服务 38 岁过日子的夫妻,还服务异地照顾父母的独生女——皮肤换了,内核不变,那条"说一句→有着落"的最短的路一直在。

# 3. ⭐ 五条第一性设计原则

这五条是从**所有调研收敛出的铁律**,不是产品经理的偏好,而是被多个独立样本反复验证顶出来的。每条给出证据链与证据强度。它们的优先级高于任何单点功能——任何功能(含未来迭代)上线前逐条自查,命中即打回。

## 原则一:去 KPI 化 —— 加法叙事,非比率叙事

**铁律**:全站文案禁用"办妥率排名 / 完成率 / 效率 / 成绩单";进度%的语言包装永远是"经验条 / 着落感";数字永远做**加法**("我们已经拉勾 27 次""在一起 N 天没换主人"),不做**比率**("本周兑现率 43%")。同一个数据,叙事方式决定了它是回忆还是考核。

**证据链(5 个独立样本命中,证据强度:强)**:
- **拉勾 6-persona**:6 位里有 4 位(苏念、方悦、林墨、夏一果)明确提到"兑现率"这种百分比容易被过度解读成 KPI / 考试分数,尤其在关系还不稳固的暧昧转正期和缺乏自信的第一次恋爱者身上杀伤力最大。原句:"万一'兑现率'常年很低,这个数字摆在那,反而变成新的吵架导火索,从'他忘了'升级成'系统证明他就是不上心'。"
- **念想 6-persona**:6 人里 3 人独立提到对百分比统计的警惕(韩静:数字会被拿来比较;郑小北:出差方结构性吃亏会被放大成情感负债;沈知微:被家人看到引发不必要联想)。
- **女王 6-persona**:Persona 1、2 共同担忧"护驾达成率 / 护驾积分"被产品自己"喂养"成关系健康度的量化审判工具,制造"你才 60% 忠诚"这种反效果压力。
- **修勾 6-persona**:Persona 4、5、6 共同提到忠诚度 / 听话度一旦长成可比较、可攀比的数字,就重演"记账式婚姻 / 绩效考核"的老红线;郑允浩(讨好型人格)会把"办妥率"当成自我价值的唯一衡量标准。
- **舰队 + 社媒**:MiniMax 红队(feedback-5)明确"「三态 + 进度 %」本质上是 Jira / Slack 工单系统套在爱情上,一条截图就能上热搜:'我以为谈恋爱,结果领了个工单 App'";Kimi 怀疑者(feedback-3)林可欣:"'在办(带进度 %)'是最毒的设计……你给我加个进度条,等于逼我表态。"美国 GTM 更是把这条上升为法理:MIT Technology Review 专题、Reddit r/TwoXChromosomes 5,496 赞帖里"manager / boss"是骂人的词。
- **PRD 已落地**:PRD §4 红线 3 明确"不做 KPI",并把 README 路线图里"排名与奖励机制"一度废弃(后经 2026-07-12 老板拍板以 opt-in、默认关闭、含"赎回 / 补过"机制的形态恢复,但"不做默认公开比较、不做监控型数据"的约束不变)。

**执行落地**:统计卡从"我的念念办妥率"改叙述式回顾;进度%在念想模式对适老 / 长辈用户直接隐藏百分比改文字"在办呢";"办妥率周报"若做,仅本人可见、产品内无入口供对方查看、禁用"率 / 排名 / 比上周下降"式语言。

## 原则二:反监控 —— 无已读、不做证据本

**铁律**:永不实现、永不展示已读状态 / 已读时长 / 在线状态 / 地理位置 / "未响应 X 小时"。进度是"看"不是"催",提醒的升级只面向执行方本人。历史记录无逾期红字,办砸 / 过期的事默认不进任何汇总、档案、周报正文,不提供"导出对方失败记录"类能力。**这三项(已读时长、在线状态、地理位置)在数据库 schema 层面就不设字段,杜绝"先存着以后用"。**

**证据链(社媒两平台收敛 + 全链路验证,证据强度:极强)**:
- **小红书**:iPhone 情侣模式 / 定位类产品高赞评论一边倒"坐牢模式"(89 赞)、"分手模式"(72 赞)、"感觉被监控";而"无已读、单向、私密、有温度"的轻信号(啾啾 / 见字、Bond Touch 的 buzz)获得纯正向口碑。原声需求说明书:"不想开始聊天、不想解释、不想等待回应""一个按钮,点一下……没有已读、没有聊天框、没有压力"。
- **Reddit**:一个 vocal 群体觉得情侣 App "dystopian"、score-keeping 有毒;反 App 逆反声音很响。资安视角(Kimi feedback-3 阿瑟):"任务卡会变成关系的'数字证据',分手之后被截图、被转发、被当作吵架素材。"
- **两份 persona 独立收敛**:执行方最高频卸载理由是"被监控、被翻旧账"的感觉;UC-03 / UC-12 / UC-17 / UC-20 四个"反监控设计"专项 Use Case 全部围绕"她那边永远看不到'他已读未办 X 小时'这种审判性数据"。
- **韩国生死线**:GS25 "捏手指"海报被男性社区解读、引发抵制、母公司股价当日 -2.37%;"她指派 / 他跑腿"框架对着韩国最危险的社会断层线(젠더갈등)。数据事故:Between 2025-09 误删十年照片,新进入者第一张牌必须是"数据永不丢失"的可验证承诺。
- **PRD 已落地**:PRD §4 红线 1、2 + §5.1 硬约束("永不采集、永不存储、永不展示:已读时长、在线状态、地理位置");红队自查项:用发起方账号穷举所有页面与通知,确认无一处可推断对方实时行为。

**执行落地**:F-02 免打扰与提醒分级——发起方端任何界面不出现对方已读状态 / 已读时长 / 在线状态 / "未响应 X 小时";F-12 离线时长判定仅在执行方本人端触发摘要,不落库为可查询的行为记录;分手 / 退出小家时单方可导出自己那份(伦理设计)。

## 原则三:安全阀是一等公民

**铁律**:任何带权力话术 / 角色扮演 / 强情绪的模式,安全阀不是设置项里的一个开关,而是产品的一等公民——**双人握手确认(进入)+ 零阻力退出(退出)+ 对称模式(框架)+ 敏感内容脱离皮肤(底线)**。退出机制必须比进入机制更轻;安全底线的优先级高于任何用户自定义。

**证据链(女王 6-persona + 韩国出海双重收敛,证据强度:强)**:
- **女王 Persona 1(沈亦谦)**:"一旦有人觉得不好玩了,说一句'我们正常说话',对方必须立刻切回普通语气,不许多问一句为什么……任何'确定要退出吗'的二次确认都是在给这个安全阀添堵。"
- **女王 Persona 4(阿九,BDSM-lite 圈层)**:给出最系统的四条——开启不能单人操作(必须双方各自设备主动确认,不是弹一次警告点"我知道了")、红灯零阻力(不能有二次挽留)、涉真实伤害风险话题无条件脱离角色皮肤(不能被自定义规则覆盖)、数据私密(角色扮演文本绝不泄露给家人共享云 / 公司审计 / 运营抽查)。
- **对称模式(韩国生死线)**:把"指派—验收"重构为"约定—共创"的对称设计——单向流改双向对称、UI 完全对等;"已接收 / 进行中 / 已完成"验收态改"우리의약속(我们的约定)"共同事项;主视觉从"女生指挥男生跑腿"改为两角色对等(学 Between 메리&밀크,一对不是主仆)。这不是措辞问题,是生死问题。
- **舰队印证**:MiniMax feedback-1 提出"看雷达前先签情绪安全协议"、feedback-5 红队"五模式一律做角色 A ⇄ 角色 B 对称 UI,文案必须过性别中立审核"。

**执行落地**:C 模式开启改"邀请-接受"式双设备确认;常驻"回到普通语气"按钮秒退、无二次确认;医疗 / 金钱 / 人身安全关键词强制中性呈现(schema 层不设声纹字段,天然合规);五模式全部支持角色互换(可选、非义务、不做"你们该轮流才公平"的说教)。

## 原则四:AI 只做成本,不碰感受

**铁律**:在"事"的链路上(转写、结构化、调研、提醒),AI 可以全自动;在"情"的链路上(道歉、表白、确认、惊喜),AI 只能做助教——**所有到达对方的情感表达,必须经过本人的手**。判据一句话:凡是"被对方发现是 AI 做的会让 TA 寒心"的事,一律不做。

**证据链(实证 + 红队自攻收敛,证据强度:强,含硬实证)**:
- **硬实证**:Hohenstein 等人 2023 年发表于 Scientific Reports 的研究——AI 辅助沟通确实让交流更快、更正面、双方评价更亲密,**但一旦被对方怀疑在用 AI,当事人会被更负面地评价**。AI 的帮助只在隐身时增值,暴露即负资产。这是 L3c(冲突调解)的物理定律。
- **AI 分层论证(L0–L3)**:L0 转写(地基零差异化)、L1 理解(产品心脏,第一 KPI 是修正率而非 DAU)、L2 代办(方案是"给他的底稿"不是"给她的答卷",AI 出题、他答卷、她收到的是他的答卷)、L3a 偏好记忆 / L3b 纪念日提醒(低越界高价值尽早做)、**L3c 冲突调解只做半个**(只对他单方、在他私密界面提供思路,绝不生成可直接转发的和好话术,绝不出现在共同界面,绝不主动检测"你们吵架了吧")。多数人做"AI+情侣"会先冲 L3c,那恰恰是唯一不该碰的深水区。
- **坚决不做的五件事**:不代替道歉 / 不代写情话、不伪造关心(三态按钮必须人按)、不当关系裁判、不做单方情报工具、不伪装主动。
- **舰队印证**:MiniMax feedback-1 "AI 是副驾驶,不是司机"(随便翻译官须留"原话保留"开关)、feedback-5 "AI 僚机不要替伴侣出方案,改成问题清单 + 聊天脚手架";Kimi feedback-3 阿杰"AI 僚机不是帮忙,是搶戏……那我的角色是什么?ATM 吗?"
- **通胀治理的第二层必要性**:零成本录入必然导致"念念通胀"(她一天发 20 条,他被淹没,任务卡贬值)——这是前 LLM 时代从未出现的死法。AI 不只是降录入成本的工具,还是"零成本录入时代的中央银行",负责合并同类项、优先级排序、每日摘要、对低权重念念延迟下发。没有 AI 的节流,零成本录入自己会杀死产品。

**执行落地**:L1 修正率作为 MVP 第一 KPI(直觉阈值 <15%,需实测标定);方案页署名永远是"AI 僚机为你准备",全站无"她要求 / 她布置"式措辞;三态按钮禁止任何自动确认 / 自动回执;冲突调解只在他主动求助时、仅对他单方响应。

## 原则五:数据永不丢失

**铁律**:数据永不丢失是情侣 / 关系产品的第一信任红线。软删除、本地副本、单方可导出、永不物理删除。这不只是工程要求,是可对外宣传的可验证营销承诺。

**证据链(Between 信任真空 + 社媒最致命抱怨,证据强度:极强,含血泪判例)**:
- **韩国 Between 事故(判例级)**:2025-09,DLT Partners 做服务器整理误删大量在用用户相册照片 / 视频 / 头像,无备份、彻底无法恢复,有用户损失近十年 / 1 万张照片。MAU 从峰值 120 万崩到 233,350(-80.6%)。补偿被评为"半吊子道歉"。这制造了韩国情侣 App 市场"十年一遇的信任真空"——新进入者的入场文案已经写好:"우리는 당신의 기억을 지우지 않습니다(我们不会删掉你的回忆)"+ 一键从 Between 导入 + 数据可随时完整导出。**反向:中国团队重演一次 = 出局 + PIPC 全体营收 3% 罚款。**
- **小红书最致命抱怨**:情侣空间改版"相册没了!全没了!"3000 天记录蒸发——情侣产品第一信任红线是"数据永不丢"。
- **关系资产 = 迁移护城河**:Rusbult 投资模型——两个人共同办成的一件件事沉淀成"着落档案",删掉这个 App 等于删掉两个人的一段共同历史。大厂能抄一个功能,抄不走两个人沉淀两年的共同记忆(214 件她说过的事、187 件他办成的心意、每个纪念日的具体坐标)。

**执行落地**:SwiftData 本地镜像表(App 启动读本地秒开,后台增量同步),服务端出事时用户本地仍有全量;`ee_items` 加 `deleted_at` 软删除列,永不物理删;提供"导出我的那一份"(JSON + 音频打包);iOS 文档把"数据安全"营销承诺钉在"永不丢失 + 可导出 + 不做声纹"上(这些是真做得到的),全链路 E2EE 留 v2。

# 4. 用户与人群

产品的用户结构有一个**天然的性别不对称**,这是它最大的机会,也是它最大的风险。八份"发起方"里七位是女性,八份"接收方"里六位是男性——这不是刻意设计的样本,是照着真实吐槽的原型剧本抓出来的分布。发起方要的是"我的话被放在了心上",接收方要的是"我做到了、我被看见了"——同一枚硬币的两面,都是同一个刚需:被确认。下面按五模式对应的人群、心境、关系阶段综合成画像。

## 4.1 发起方(多为女性):被"心理负荷"压垮的那个人

她承担的东西,准确的名字叫**"心理负荷(mental load)"**——那种"永远得有一个人在脑子里跑着一个项目管理器",记着家里所有该办没办的事、该提醒谁、什么时候截止。日本给它起了个更传神的名字"名もなき家事(没有名字的家务)",而这类无名家务里排第一的正是"管理"。反复出现的一句话是:"我不是要一份礼物,我是想感觉这个家是两个人在一起维护的,而不是我一个人在管理生活,他只是恰好住在这里。"(陈雨桐,26 岁,同居两年。)

她的核心痛不是"他忘了"(记忆问题),是"他漏接了"(关系问题)。她土办法用过很多:微信后面加一句"记得哦~",他回"好的"但那两个字不会变成行动;后来建了共享备忘录,他压根不主动点开,变成她一个人守着的空房子。所以她最讨厌的动态是"当那个不得不去提醒的人"——一个天真的提醒 App 会把她变成男方任务的"管理员",反而强化她最讨厌的动态。这是"念想"是最大盘子却最难切的楔子的原因。

## 4.2 接收方(多为男性):善意在场、能力失灵的"猪队友"

产品救得了"善意但失灵"的关系(意愿在、执行系统差),救不了意愿死亡的关系——这不是漏洞,是目标用户定义:**服务"猪队友",不服务"渣男"**。最典型的是大鹏(38 岁,自述 ADHD 倾向):"我这脑子真的有问题,别人一句话进耳朵五分钟后就飘走了,我自己也很痛苦很自责,越自责越焦虑,越焦虑越记不住。我需要的不是'更用心'这种道德说教,我需要的是外部结构化系统。" 对他这样的大脑,"办妥"带来的正反馈是他婚姻里可能稀缺了十年的成就感——这就是必须给男方设计并度量 Aha 时刻(注册 7 天内办妥 ≥2 件并收到 ≥1 次回响仪式)的原因。

男性用户对产品叙事极度敏感:阿杰"我寧願她直接發火,也不要我回一句'好'之後,系統自動生成一張待辦卡掛在我頭上";周涛"晚上 11 点弹通知、开会时 AI 语音大声播报'老婆让你买洗衣机',或者陈敏用它替代一切沟通,让我感觉在上班打卡,那就拜拜"。所以对男方的整个叙事必须是"僚机"不是"监工",气泡感聊天而非正式待办工具的压迫感。

## 4.4 五模式 persona 实录精选(用真实的翻车,看清真实的人)

抽象画像不如具体的疼。下面从五份 6-persona 用户研究里各选最有代表性的样本,用他们自己的翻车故事和原声,把"人群"落到肉里——这些不是脑补,是用户模拟里逐字写出来的心境。

**念想 · 韩静(32 岁,上海,双职工无娃,结婚 4 年)**:她试着用现有念响模式走一遍——下班地铁里随口说"热水器报修,周三下午在家等",这个体验是戳中的,这句话本身就精准表达了她要的:不是要老周记住,是要"这事有个着落",不用一直心里挂着。三态她最认——报修这种事她最烦的就是不知道老周到底安排了没有。但走到"办妥"弹出"办妥 ✓ 你做到了"+庆祝动效,俩人对着屏幕都有点尴尬:"热水器修好这种事,不需要被夸'你做到了',这语气像是在哄小孩收拾玩具。" 物业费交完弹"办妥 ✓ 你做到了"说实话有点想笑——"这不是什么值得庆祝的成就,是本该做的家庭责任。" 统计卡"办妥率"她反而介意:"4 年婚姻,谁都不想被一个百分比量化说你只办妥了 70%,哪怕这数字只有我们自己看得到,心理上也会不自觉地拿来比这周你比我少。" 她真正的担心一针见血:"这套系统会变成又一份需要我们共同维护的账本——如果哪天有一方不看了,反而制造新的'我说了你怎么没看'的心结。"

**念想 · 郑小北(34 岁,北京,老公一年出差 150 天+)**:对她这种情况,三态是"救命的"——她最痛苦的不是老公不做事,是完全不知道他在客户那边有没有看到、有没有安排,现在能看到卡片"收到"哪怕他还没空处理,心里至少有个底,不用一遍遍打电话追问显得像查岗。但她极其在意"办妥率":"老公常年不在家,他能在场办妥的事本来就比双职工同城家庭少很多,如果这个数字被解读成'你看你又没办成几件',会放大他本来就有的出差愧疚感,变成压在他身上的一份新的情感负债。" 这是"去 KPI"原则里"结构性劣势会被放大成情感负债"最锋利的一条证据。

**拉勾 · 苏念(19 岁,长沙,校园情侣 8 个月)**:"戳我!非常戳!我们这个年纪就是喜欢这种'小学生过家家'式的仪式感,拉勾上吊本来就是我们俩的梗——他手机壁纸现在就是我们拉勾的九宫格照片。'拉钩成立'四个字比冷冰冰的'已收到'可爱一百倍,办妥了盖一个'说到做到'的章,截图发朋友圈那种。" 但她也精准说出了这个模式的原罪:"怕它变成'任务清单式的爱情'——万一'兑现率'常年很低,这个数字摆在那,反而变成新的吵架导火索,从'他忘了'升级成'系统证明他就是不上心'。" 六位 persona 的担心几乎全部指向同一个核心矛盾:仪式感的魅力在于"自发",一旦被系统架上"流程"和"数字",就有滑向"任务 / KPI / 催命"的风险。

**女王 · 苏晚(30 岁,创业公司 COO,备婚中)**:她带 40 人团队,回家最讨厌还要"哄"、还要"商量"。视觉她给满分(黑金、火漆、"圣旨 No.043"编号感),但话术包整体是"主人养宠物"框架("汪!遵命""摇尾巴中""求摸头"),而她男友属性更接近"能干但需要指挥的下属","如果只能用这一套词,我会觉得这不是我们,就像给一份西装找错了内搭。" 这是女王必须拆双支线的核心证据。她还有一句极重要的边界:"我不希望这套东西被扭曲成真实的权力不对等……真正大事(买房、要不要孩子)不该套进这套语气里讨论。"——安全底线不能被角色皮肤覆盖,由此而来。

**女王 · 阿九(27 岁,与男友同为 Kink/BDSM-lite 圈层,3 年)**:圈内固定 S/M,他们借用已经很成熟的"红黄绿灯"安全词系统。阿九给出了全项目最系统的安全设计:"权力交换必须建立在充分沟通在先、随时可撤回、绝不延伸到现实决策权这三条上……第一,开启不能是单人操作,必须双方各自在自己设备上主动确认;第二,红灯必须是零阻力的,不能有'确定要退出吗''再想想'这种二次挽留式弹窗,那是对信任的破坏;第三,涉及真实伤害风险的话题(生病吃药、意外、真实冲突)必须无条件脱离角色皮肤,这条不能被自定义规则覆盖。" 安全阀一等公民原则的骨架,几乎全来自这一位。

**修勾 · 林薇(25 岁,苏州,管男友叫"我家狗子")**:"毒舌互怼很对我的路子,'我家狗子'本来就是带调侃的爱称,任务卡变成'汪,别催'完全符合我对他'拖延又嘴硬'的心理预期。但纯怼久了会腻,我需要'暖'的落点——比如摇尾巴反馈这种视觉化的东西,才让我真的觉得这是我们俩的狗,不是客服机器人在处理工单。两种调性能并存,关键是节奏:平时怼,重要事暖。" 她指出真正伤感情的不是毒舌话术本身,是"当众"——把"通缉令"截图发朋友圈,玩笑一旦被外部看见就变成社死。

**修勾 · 周敏(39 岁,三甲医院主治医师,结婚 11 年)**:"我这个年纪已经过了'互怼图新鲜'的阶段,纯毒舌吸引力有限……真正打动我的是'小狗陪伴感'——那种很多年了这只狗还在的稳定感。" 结婚纪念日老公发"狗狗汇报:11 年,一次没换主人"——"这种傻兮兮的忠诚宣言比钻石更让我感动,因为他平时不是会说甜言蜜语的人,玩梗给了他一个说肉麻话不尴尬的出口。" 这句"玩梗给了他一个说肉麻话不尴尬的出口",是"毒舌是壳、小狗是核"最凝练的注脚。

**爸妈 · 周建国(65 岁,武汉,心脏支架术后,儿子在新加坡)**:"儿子每天早晚视频问诊式地追问血压多少、有没有胸闷,时间长了我开始有点抵触,有次直接说我又不是废人,你天天这样问我,我压力比生病本身还大。" 他把界限说得极清楚:"这个 App 不能让我变成一个被 24 小时监控生命体征的病人。术后我已经因为疾病本身丧失了不少自主感,如果连日常提醒都做成实时血压心率仪表盘给儿子看,那种被时刻审视的感觉比生病本身更让人沮丧。它应该做的是'提醒有没有被看到'这一件事,仅此而已——这样我按一下按钮,是我自己在确认'我还好',而不是设备在替我汇报'他还活着'。" 这就是"安静侦测不能演化成生命体征仪表盘"的原声出处。

**爸妈 · 高秀云(58 岁,西安,反过来催儿子相亲的妈妈)**:她把每天想说的相亲催促像连环 call 一样发给儿子,后来儿子直接不读不回,母子冷战。她的领悟极有价值:"我把'催婚'这种带评判性质的念叨,和'记得吃早饭'这种纯粹的关心混在一起发,本质上是在用关心当外壳去施加压力……儿子看到我发的所有消息都变得抵触,因为总怕后面又跟着一句催婚。" 这是"健康提醒与催婚必须分渠道"最直接的证据——一旦混发,接收方会对整个系统的所有消息产生防御性预判,连累本该被认真对待的关心。

## 4.5 被低估的外溢人群(社媒抓到的真实需求)

社会聆听发现几个"没想到、最值钱"的人群外溢(`SOCIAL_LISTENING.md` §2 Q2):① **闺蜜 / 朋友 / 家人,不止情侣**("两个好闺蜜都很忙,不好意思打扰,但真的很想她们""只能一对一吗,我想对我闺蜜发");② **异地照护的成年子女**(现有手机日历失效因父母"不看",被迫自建来电提醒——最高意图最被忽视的一群);③ **多照护者的"到底吃没吃"核验与留痕**(现有 med app 全"assume one user",这是空白);④ **"自己连自己"/单身自我记录**;⑤ **异地恋 / 军营的"想念一瞬即刻传达"**(愿为硬件花 $60–160/对,但是"愤怒的买家",硬件不靠谱)。这些外溢证明底层"非对称双人轻协作 + 情绪化反馈"不是情侣专利,是横向拓展的天然接口——但节奏上要"先把情侣主线闭环跑通,再放外溢"。

# 5. 破圈策略

## 5.1 为什么第一版可能靠"修勾 / 冲突感"破圈,而非甜蜜

**核心判断(社媒 + 舰队高度收敛)**:第一版押"毒舌修勾"作破圈引擎,其它四个模式当留存和扩展用。理由是一条社交平台的物理规律——**你的产品天然带"冲突感",而冲突感才是社交平台货币。甜蜜是消费品,毒舌是创作素材。** 抖音 / 小红书上 99% 的情侣内容都在卷"他好宠我",用户已经审美疲劳;给他们一个可以截图、可以挂人、可以 @ 对象的口子,流量才会自己来。

证据强度足够:
- **社媒双平台确凿**:"小狗 / puppy"情结在中英两个文化里都独立成熟,是最强传播符号。中文"小狗 / 犬系 / 修勾"单篇破 3006、2585 赞,2025-11~2026-01 仍高频更新;英文"golden retriever boyfriend / good boy"已是稳定词汇(7.3 万赞级帖)。两平台都自带去污名化自觉(中文"不是舔狗 / 不是调教",英文强调 compliment 与 consent)。
- **舰队 MiniMax feedback-2 的斩钉截铁**:"第一版押毒舌修勾,其它四个都别当流量入口。念想是伪需求真留存(别砍,只是别指望它出圈);拉勾勾是创始团队最爱但最容易死的选项(市场上所有情侣 App 默认就是这个调性,没有差异化);毒舌修勾是王者选项。"
- **年轻情侣 persona 印证(Kimi feedback-4)**:修勾模式"太会截图了,这种自带 meme 属性的内容最破圈",会带 #电子狗链 #任务卡遛狗 话题——但也"最危险,只适合双方都心知肚明的'嘴贱情侣',否则就是下头 app"。

**但必须清醒的三条反直觉提醒**(MiniMax feedback-2):① 别把"双向"当卖点吆喝(对非用户是抽象词,卖点应具象到"你家的修勾今天上岗了吗");② 硬件挂坠现在别宣传(它是 LTV 工具不是获客工具,DAU 站稳再软植入);③ 第一版做减法是出路——先让 10w 人用"修勾 + 女王 + 拉勾"三件套跑通,出圈了再叠层。

## 5.2 三个可执行病毒点子

以下三个来自 MiniMax 增长视角(feedback-2),均已具象到可落地机制:

1. **「修勾日报」可截图卡片**:每天 22:00 自动生成一张卡片("今日挨骂 4 次 / 叼飞盘成功 1 / 被叫'小狗'7 / 今日称号:被驯服的小柴")。爆款公式 = 列表 + 数字 + 反差;卡片底部放"分享给主人,看 ta 怎么回"按钮=裂变钩子;周报 = 一次安装 7 次分享机会。**红线校准**:日报的"KPI"必须做成情趣玩具而非驯化工具,被评的一方可一键脱圈("赛博项圈"开关),"拆家"这类把"惹我生气"和"主观不服从"混为一谈的指标要慎用——这与原则一、原则三直接挂钩。
2. **「你的 TA 是哪种小狗」人格测试**:基于前 30 天对话给 TA 出一个"犬种人格"("边牧型修勾:智商在线但不服管")。这是经典的"MBTI 式社交货币"(Spotify Wrapped、网易云年度报告验证过,关系类总结卡分享率是普通内容 5–10 倍),结果会变成情侣互相 @ 的工具(UGC 自生产),狗品种 = 现成视觉资产。一键生成小红书九宫格模板降低分享摩擦。
3. **「15 秒修勾变身」抖音语音挑战**:话题 #我家那位是修勾怎么证明,引导"录一句日常需求→修勾模式回应→截屏生成 15 秒视频"。反差是抖音算法最爱的钩子,双人内容双人转发(买一送一)。配套:内置 5–6 个"修勾常用回复模板"配音;找 3–5 个中腰部(50w 粉)情侣博主开箱(别找大 V,中腰部 ROI 高 5 倍);第一周投 100 个素人素材做样子。

## 5.3 "先打基建、后放特殊模式"的节奏

破圈不等于第一版就把所有火力压在修勾上,而是一个有先后的编队:

- **第 0 阶段(地基)**:念想 + 拉勾把"说一句→有着落"的基建跑通,把男方 Aha、修正率、双端留存这三个数字盯住。念想是"App 30 天不删的关键"(留存),拉勾是解释成本最低的"承诺仪式"(拉勾这个跨文化承诺手势 0 秒讲清"这个 App 是干什么的")。
- **第 1 阶段(引爆)**:放毒舌修勾作首个可控破圈引擎,配三个病毒点子 + 中腰部博主 + 素人素材冷启动;女王作为第二传播模式跟进(但需三处措辞校准 + 双人握手 + 严格脱敏)。
- **第 2 阶段(扩圈)**:放硬件信物(七夕 / 圣诞众筹,软植入而非硬广)与家人版(独立界面入口)。
- **贯穿始终**:任何一个模式在破圈时都必须先过一遍五条第一性原则的自查,尤其是原则一(去 KPI)与原则三(安全阀)——修勾日报的数字、女王的权力话术,恰恰是最容易翻车的两处。"谁先做出'为弱势方留一票否决权'的产品,谁就能活下来"(MiniMax feedback-1)。

# 6. 市场与商业化

## 6.1 市场规模与定位难点

念念落在三个市场的交集,这是机会也是定位难点:情侣关系 / 情感记录 App(恋爱记、Between、Paired、Agapé)、双人协作 / 共享待办(Cupla、滴答清单、微信自带提醒)、情侣硬件 / 信物(Bond Touch、Lovebox)。全球"情侣关系 App"市场,不同报告口径差异很大(2025 年估值从 $0.854B 到约 $2B 不等),量级结论是:**十亿美元级、双位数增长(CAGR 约 12–15%)的利基市场,不是百亿级大赛道**。中国盘子是千万级到亿级用户(恋爱记一家 3000 万注册),但——最重要的一句实话——**用户获取被验证过,付费模式没被验证过**。

竞品的本质差异一句话:所有竞品要么做"聊 / 记 / 甜"(恋爱记、Between、Paired),要么做"冷冰冰的清单"(滴答、Cupla)。念念做的是第三条路——**关系里的责任闭环**:她说的每句话有去向,他办的每件事有凭证,AI 把"她随口一提"翻译成"他能直接执行的方案"。竞品没有一个在"忘了→吵架"这个具体故障点上给出协议级解法。

## 6.2 为什么竞品卖"甜"、我们卖"作数"

因为"甜"没有明确的失败成本,而"忘了→吵架"这条链路有:

- **明确的失败成本**:忘了纪念日 / 承诺 → 争吵 → 冷战,单次冲突的情绪代价远超一年的订阅费。付费买"不再因为这个吵架",锚点清晰。
- **明确的责任方与购买方**:女方是痛点感知者与安装发起者,男方是执行者与"被拯救者"——男方为"让我别再挨骂"付费的意愿可能比女方更强。
- **高频且可验证**:不是模糊的"增进感情",而是"这周三件事,办了两件"——效果可被双方看见,续费理由自然产生。

这就是为什么付费假设必须钉死在"着落"上。**"甜"证明了情绪共鸣能带来下载,但带不来付费**(恋爱记 3000 万用户仍靠"云宝养成 / 攒钱打卡"找收入,商业化屡遭吐槽)。第一天就收费:AI 僚机免费几次 / 月,¥98/年双人共享订阅(对标 Agapé 双人年费 £47.99,证明"双人一份订阅"是被验证的定价结构)。**第一年这不是一门"收入生意",是一门"验证生意"**——真正的 KPI 是付费转化率是否 ≥5%、次年续费意愿、以及"因为这个 App 少吵了几次架"的定性证据。

## 6.3 情绪消费大盘:我们长在靶心上

商业化踩在万亿情绪消费的盘子上:中国情绪经济 2024 年已是万亿级(艾媒口径 23077.67 亿,取量级);《2025 Z 世代情绪消费报告》(Soul × 上海市青少年研究中心)显示年轻人**月均 949 元**为情绪价值买单,付费动机 Top3 是价值契合 37.6%、自我疗愈 34.2%、**仪式感 32.3%**,而**近 8 成青年认为情绪价值主要来自"亲密关系中的理解和回应"**——这一条直接命中念念的产品定义(回应 = 三态有着落)。**我们不是在蹭情绪消费,我们长在它的靶心上。** 月均 949 元的情绪预算里,一条挂链(99–599 元)或一年订阅(¥98)完全放得下。

泡泡玛特是可抄的商业范本:FY2025 营收 371 亿、经调整净利 130.8 亿、毛利率 72.1%,六大 IP 收入均破 20 亿,证明单一审美吃不下全部人群、但矩阵可以;72.1% 毛利率证明情绪溢价能把硬件毛利做到接近软件。**可抄的是"多情绪人格矩阵 + 随身展示 + 上新驱动复购",不可抄的是"不确定性抽赌"**——念念卖的恰恰是确定性、着落感,与赌徒机制方向相反,盲抽 / 隐藏款列入设计禁区。

情绪消费的分赛道证据进一步印证"上新 × 收集 × 展示"是复购结构本身而非营销选择:谷子经济(二次元周边 / 痛包)2024 年 1689 亿元同比 +40.63%,徽章"吧唧"几元到上百元、痛包整体数百到上千元,复购逻辑是"按作品 × 角色 × 款式无限上新,痛包=把爱意外挂展示,买的是身份标签";Jellycat(毛绒陪伴)2024 天猫 + 淘宝销售额 6.2 亿同比 +84.7%,巴塞罗熊 189–1299 元,成年人买"过家家式陪伴",门店"打包表演"把交易变仪式;寺庙经济手串数十到数百元,买的是"心理确定感 / 祝福外包"。所有赛道复购都靠"上新 × 收集 × 展示"不靠单品迭代,而泡泡玛特会员复购率 2019 年 58% → 2024 年 49.4%(近半会员一年内回购但随规模缓慢下滑,情绪消费的复购要靠持续上新)。这三条对念念的硬结论:"亲密关系中的理解和回应"是青年情绪价值第一来源(近 8 成,Soul 报告),念念长在靶心上;月均 949 元的情绪预算里一条挂链 / 一年订阅完全放得下,仪式感(32.3% 付费动机)对应纪念日限定款、自我疗愈(34.2%)对应"办妥回响"的正反馈;系列化不是营销选择,是情绪消费的复购结构本身。但泡泡玛特的反面证据也要看:财报后股价单日跌 20–30%,市场担忧 Labubu 依赖症(单 IP 占比从 23% 飙到 38.1%)——矩阵的意义正是对冲爆款生命周期,但矩阵本身救不了"没有下一个爆款",系列化要留孵化机制不是一次性铺满。

## 6.4 三条腿的收入结构:订阅 + 硬件 + 皮肤

1. **订阅(主推)**:AI 僚机 + 高级提醒 + 未来家庭版升级,双人共享一份 ¥98/年,把付费绑在产品唯一性("着落")上。MiniMax 给出三档校准方案供参考——心动档 ¥68(首年特惠,25 岁以下 / 学生,次年跳回 ¥98)/ 陪伴档 ¥98(主推,不打折避免锚定塌陷)/ 珍藏档 ¥198(礼物 / 仪式感,客单敏感度最低);AI 免费额度用"5 次终身 + 配对成功再送 2 次"做网络效应钩子。
2. **硬件信物(第一年末试水)**:挂链定价锚"信物 / 礼物"(众筹双件装 ¥299–349、早鸟 ¥249),不锚"蓝牙配件"。走"众筹信物"路线,不追硬件利润,追的是"信物化"带来的留存与送礼获客——硬件当获客成本(CAC),把硬件做成微亏损入口,换取 1–3 年订阅确定性(方案 B「三年之约」¥328 锁三年)。
3. **皮肤 / 限定(辅助,绝不做主力)**:纪念日限定款、刻字序列号、办妥彩蛋,学泡泡玛特"上新 × 收集 × 展示"复购结构,坚决不抄"盲抽 / 隐藏款"赌徒机制(方向与"着落感"相反)。限定皮肤 ¥18–28/款绑定节日(女王节 3.8 / 修勾日 4.28 / 双 11)制造"错过等明年"。

## 6.5 商业化的三条清醒判断

- **付费意愿最高的模式判断有分歧**:MiniMax 押"拉勾勾"(恋爱前中期钱热最高、客单价弹性最大、名字自带流量),但同时反驳自己——**拉勾勾 LTV 最低**(同居 / 结婚后立刻"过期"),真正的长跑选手是"毒舌修勾"(一旦变成两人的沟通方言就戒不掉)。综合策略:**拉勾勾做获客漏斗(爆发)、念想做基础盘(留存)、毒舌修勾做订阅钩子(付费深度)、硬件做情感天花板(品牌 + 估值)**。
- **家庭版可能是第一年收入主力但不是主轴**:MiniMax 定价测算里家庭版(爸妈,子女代付,孝心版 ¥198 / 守护版 ¥398)占第一年营收 51.5%——B 端代理决策模型客单价显著高于 C 端。但用户研究强烈警告它"用错了容器",所以商业上可观、产品上必须独立界面。这是"收入诱惑"与"产品红线"之间需要老板拍板的张力点。
- **双订阅陷阱与 AI 成本**:情侣 App 付费意愿天然是双倍阻力(双方都要付=单价 ÷2),必须做成"一人订阅、另一人白嫖解锁",否则付费率锁死在 5% 以下;AI 僚机的 token 成本在超过 100 万 DAU 后会吃掉订阅收入的 40%,早期必须用小模型 + RAG 控成本。第一年收入三档估算:保守 ≈¥1 万 / 中性 ≈¥14 万 / 乐观 ≈¥106 万——中性档连一个人工资都不够,再次印证"第一年是验证生意"。

## 6.6 系列化:学泡泡玛特的三层结构,不是二选一

多系列战略不是"模式切换 vs 独立子品牌"的二选一,而是学泡泡玛特的三层结构:① **公司 / 产品内核一个**(账号、关系链、三态协议、AI 僚机、硬件 API 全共享,相当于泡泡玛特的品牌 + 会员 + 供应链);② **系列即"IP 人格"**(拉勾勾、念想、家人系是对外叫得响的系列名,各有硬件形态、界面主题、文案人格、定价带,对外看是不同产品,对内是同一内核的皮肤 + 参数,相当于 Labubu / SKULLPANDA / 星星人);③ **模式切换只发生在同系列内部**(比如拉勾勾内提供"甜蜜 / 契约 / 大女主"三种语气模式作为低成本 A/B 试验田,哪个跑赢再升格为独立系列);④ **唯一必须物理隔离的是家人系 / 长辈系**(交互范式大字、语音、底座硬件与情侣端差异太大,共内核、不共界面入口)。

首发顺序建议(EMOTION_ECONOMY 推理):第一发念想·中年派(现有 v1.0 产品的人群本体,不需新开发,只需把"系列"叙事立起来)→ 第二发拉勾勾·年轻派(传播引擎,挂件外显 = 自传播,七夕众筹天然适合此系列打头阵)→ 第三发长辈系(横向拓展中需求最真的一支,与中年派用户是同一批购买者,交叉销售路径最短)→ 第四发亲子系(需求真但合规与反监控难度高,晚做)。复购结构照抄"上新 × 限定"、坚决不抄"抽赌"(纪念日限定款、刻字序列号、办妥彩蛋做仪式感对应 32.3% 青年为仪式感付费,隐藏款 / 盲抽机制与"着落感"心智相反列入设计禁区)。横向拓展总排序(判断标准:与"为放下心付费"核心情绪同构度 × 与三红线冲突度 × 竞争空档):子女→父母吃药提醒(最扎实,MedMinder $35M 融资验证子女付费,国内智能药盒仍是空档)>> 亲子(妈妈给初中生,需求真但形态危险,初中生视角 90% 概率被体验为监控)> 护工-家属(挂靠子女→父母的企业版)> 助理-老板(不做,会撞钉钉 / 飞书)≈ 商务协作(不做,无情绪溢价)。定价对齐情绪预算而非硬件成本:年轻人月均情绪消费 949 元、Jellycat 单只 129–1999 元、泡泡玛特毛利 72.1%,挂链定价锚"信物 / 礼物"(99–599 元分系列)不锚"蓝牙配件",软件付费继续钉死在"着落"订阅、不做皮肤依赖型收入。

## 6.7 竞品拆解:五座墓碑与一句判词

理解这个品类,先读墓志铭。2011–2017 年全世界至少五家团队做过和我们几乎一样的事,这份名单是这个品类过去十五年最有说服力的部分:

- **Pair / Couple(美,YC W12,2012)**:情侣私密聊天、thumbkiss、清单、日历,2013 并购英国 Cupple 改名 Couple,2016-02 被 Life360 收购后关停下架。
- **Avocado(美,ex-Google 团队,2012)**:共享清单 + 共享日历 + 提醒 + 私密相册,2017-02 因缺资金关停。**这是决定性证据**——功能定义上它就是一个没有 AI 的念念,想法一模一样、团队来自 Google、媒体口碑好,仍然死了。这是一个干净得罕见的反事实实验:当年缺的既不是需求也不是产品想法,缺的是把"她的话"变成"结构化任务"、把"任务"变成"可执行方案"的那个劳动力。
- **Between(韩,VCNC,2011)**:情侣聊天 / 相册 / 纪念日,巅峰自称 3500 万对情侣、DAU 曾破 100 万,是这条赛道里最成功的一家——却以创始团队弃赛(朴宰旭转身去做网约车 Tada)、公司七年被转手四次、2025 亲手炸掉自己十年护城河(误删相册)告终。它的死法与恋爱记一模一样:变现只能靠贴纸 / 周边 / IP 电商,情绪共鸣带来下载,带不来付费。
- **小恩爱(中,2012)**:情侣空间 / 打卡,2014 拿 $10M A 轮,靠"恩爱银行"打卡返现拉活,2019 年末爆雷遭千人投诉,月独立设备九个月 -39%。
- **恋爱记(中,2014)**:未死,但 3000 万注册仍靠"云宝养成 / 攒钱打卡"游戏化道具找收入,商业化屡遭差评。

现存对标里最近的四个:**Cupla**(情侣共享日历 + 待办,50 万+ 情侣,但骨架是"日历工具 + 情侣皮肤",没有"她说→他办"的方向性协议、没有催办 / 着落凭证,AI 只做约会点子不做方案执行);**Paired**(美,品类第一,$14.99/月 / $74.99/年,800 万下载、100 万+ MAU,纯内容订阅无工具属性,不解决任何具体的"事"——它证明了美国情侣 App 唯一跑通付费的一家,但走的是"关系练习"内容路);**Cozi**(家庭日历,2000 万+ 注册,90% 用户是有孩子的女性——这本身就是"数字化心智负担"的实锤);**Milo**(YC + OpenAI 直投,GPT-4 驱动主打 invisible load,语音 / 截图丢进去自动变日程,功能上与我们最像的美国产品,必须持续盯)。

**一句判词**:所有竞品要么做"聊 / 记 / 甜",要么做"冷冰冰的清单"。**卖甜的证明了情绪共鸣能带来下载带不来付费;卖清单的没有情绪与关系语义,给对象派任务像上级派活。** 念念做的是关系里的责任闭环——竞品没有一个在"忘了→吵架"这个具体故障点上给出协议级解法。历史的教训不是"现在时机到了所以稳了",是"现在及格线终于可以被够到了,而当年连及格线都不存在"。Avocado 想做的那件对的事,它输给了时代;我们不是比它聪明,是比它晚生了十年——晚在了那个劳动力刚刚被造出来的年份。

## 6.8 舰队十份反馈的收敛与分歧

十份舰队反馈(Kimi×5 + MiniMax×5)是本项目的"红蓝队攻防"——独立视角挑刺、举反例、给反面意见。以下把它们的收敛点(证据强)与分歧点(需老板判断)分开列,不做中立罗列。

**高度收敛的六条(可直接当铁律)**:
1. **老夫老妻能用,但用法完全不同**(Kimi-2 三对夫妻共识):热恋要"甜蜜仪式",老夫老妻要"责任闭环";最核心痛点不是"记不住"而是"以为对方会做";念想≈默认答案,拉勾勾偶尔可用,女王 / 修勾只适用于已有私密默契的夫妻且绝不能在公共场合弹通知。判词一针见血:"这款产品对老夫老妻有没有用,取决于它是否愿意承认自己不是'恋爱神器',而是'家庭协作系统'。"
2. **异国恋刚需是"异步 + 反监控 + 静音防水硬件"**(Kimi-1 三对异国恋):任务卡 + AI 僚机是刚需,但挂坠如果不做"时差勿扰"就是干扰;共同雷点是订阅制、外放声音、缺时区勿扰;三对分别适配念想 / 拉勾 / 修勾(印证五模式是光谱)。
3. **爸妈模式"摸到真痛点但用错容器"**(Kimi-5 三个真实场景):老人能用但"不是自然会用,是被要求用";"大字"只是给子女展示"我考虑到了老人",不是给老人真正使用;"关心变监控"的界限在"是否把对方的自主生活变成必须汇报的 KPI",而"爸妈听话"这四个字直接颠倒代际权力关系;结论"老人要的不是'听话',是被听见"。
4. **怀疑者的三记重拳全部命中红线**(Kimi-3):"在办带进度%"是最毒的设计(逼人表态)、任务卡会变成关系的"数字证据"、语音优先=生物特征优先(资安敏感者的过敏)——这三条恰恰是我们五条原则(去 KPI、反监控 / 不做证据本、AI 只做成本)存在的理由,把怀疑者的攻击当成设计约束吸收。
5. **第一版做减法是出路**(MiniMax-2 + Kimi 多份):第一版功能越多新人越懵,先让人用"修勾 + 女王 + 拉勾"三件套跑通,爸妈 / 念想别当流量入口;押毒舌修勾破圈(甜蜜是消费品,毒舌是创作素材)。
6. **最容易翻车的不是技术,是把人之常情量化后产生的权力失衡**(MiniMax-1 + MiniMax-5 红队):进度条把爱人变成工单、五模式踩遍性别政治雷区、分手后 App 是核弹、硬件从浪漫道具滑向监控 / 家暴入口、微信 / iOS / 小米三方夹击护城河只剩 6 个月——"谁先做出为弱势方留一票否决权的产品,谁就能活下来"。

**需要老板判断的分歧点**:
- **付费意愿最高的模式**:MiniMax-3 押"拉勾勾"(钱热最高)但又自我反驳"拉勾勾 LTV 最低、真正长跑是毒舌修勾";这与"念想才是留存地基"的判断并存,需要一个明确的"获客 / 留存 / 付费 / 品牌"四模式分工。
- **中日韩三个市场几乎是三个产品**(MiniMax-4 的犀利总结):"日本男人要的是被公平对待,韩国男人要的是被看得起,中国男人要的是被理解;同一款 App,三个市场三种卖法。" 日本是中老年人的真需求、韩国是 20 代年轻人的真需求——这两个市场打法完全不同,几乎可以当两个产品做。这直接影响出海资源如何分配。
- **8 个"尖叫功能"的取舍**(MiniMax-1):默契值盲区雷达、女王契约沙盒、修勾日报、想念无声心跳、关系体温计、随便翻译官、吵架冷静期——PM 自评 5 个推、2 个灰度、1 个先灰度实验;每个都踩在"亲密关系产品化"这条线上,是否做、做到什么程度需要老板在"传播爆点"与"红线风险"之间拍板。

**舰队给的最锋利一句(值得刻在墙上)**:"这个产品最大的翻车点,不是功能不够好,而是它对'亲密'的理解太工程师了。把'想念'做成按钮的那一刻,你就输了一半。"——这句话不是否定,是提醒:五条第一性原则存在的全部意义,就是不让这句话成真。

## 6.9 硬件差异化:情侣间的轻量传话信物

硬件竞品(Plaud、Anker 录音豆、钉钉 A1、讯飞、Limitless、Omi)全都偏商务会议,四个结构性原因也正是我们的空档:① 转写 / 摘要是持续云成本,只有"效率有价"的 B 端 / 职场人群愿意付月费($15.99–19/月),消费 / 情感场景撑不起订阅所以没人做;② 会议是"长录音"刚需(30 分钟以上才凸显 AI 摘要价值),厂商自然向长续航、多麦克风、远场拾音堆料,10 秒钟的一句话留言在他们的产品逻辑里"不值得做";③ ROI 故事好讲("省下写纪要的一小时"容易成交,"让 TA 记得你说过的话"无法进 PPT);④ 路径依赖(讯飞从速记、钉钉从办公协同、Plaud 从通话录音起家,基因都是"记录已发生的信息",没有一家的原点是"向另一个人传递")。

**全行业都在做"录音→转写→给自己的纪要",没有人做"录音→转写→到 TA 的清单"。这是关系型(dyadic)语音硬件的空白。** 我们的一句话定位:**按一下,话就到 TA 清单里。** 只做一条流水线(录音≤60 秒→云端转写→生成一张念想卡→落进 TA 清单→TA 手机轻推送),明确不做会议纪要、长录音、说话人分离、多语言翻译、always-on、屏幕——每一项"不做"都同时省下 BOM、续航、隐私风险和解释成本。技术门槛反着用:竞品的护城河(麦克风阵列、远场拾音、长续航、机内转写)在 0.5 米对嘴说话的场景全部失效,单麦 + 300mAh 就够,省下的 BOM 花在材质、造型、开箱仪式感上(这是效率厂商的组织能力做不出来的);订阅逻辑反着用:一句话 10 秒音频的转写成本约为会议场景的 1/100,免费额度可以真·无限感,砍掉用户最恨的"分钟数墙";购买理由不同赛道:对标的不是录音笔,是情侣手环 / 异地恋礼物(礼赠市场定价宽容,双件装天然合理)。

**MVP 形态**:「念想磁吸」MagSafe 挂坠二合一(圆形小坠,背面 MagSafe 磁环吸手机背面、顶部留孔穿挂绳变手机挂链,一个模具吃下两种形态,"吸在手机上"当默认态解决"忘带"、"挂起来"当情感态)。规格:BLE 5.x 单麦 PTT(按住说话≤60s)+ RGB 呼吸灯(对方收到 / 给你留言两种灯语)+ 磁吸触点充电,重量≤15g,目标 BOM ¥55–75。技术路径 MVP 走 BLE + App 代传(功耗极低、无配网、隐私链路可解释,音频经用户自己手机上传),Wi-Fi 直连只留给三期桌面摆件。**隐私红线**:实体录音键按住才录(从交互上排除 always-on,这是与 Limitless"记忆假体"划清界限的核心)、录音灯示固件级绑定不可软件关闭、不做隐蔽外观(国内对"伪装式窃听窃照器材"有刑事红线)、音频转写后默认删除。三步走:第一步(0–3 月)零硬件验证(Action Button / Watch / AirPods 适配 + ¥10 蓝牙按钮灰度,通过标准实体入口人均日留言≥App 内按钮 2 倍)→ 第二步(3–9 月)第一款信物量产 → 第三步(9–18 月)系列化成家(拉勾勾对装 + 大女主项圈 + Wi-Fi「念念小屋」桌面摆件)。

# 7. 出海战略

## 7.1 最大出海资产:拉勾仪式中日韩美同源

核心假设成立,而且比想的还强:拉勾仪式在中日韩美**同源**——不是"巧合地相似",是**从日本江户吉原的「指切り」向外扩散的同一个仪式**。游女为向恩客立誓真的切下左手小指送人(心中立),后被庶民和儿童模仿退化为"勾小指"游戏。日→韩(韩国学界口径一致源自日本 유비키리)、日→美(英文维基把 pinky swear 源头也归到日本)。这给品牌一个真实、可考证、跨国的历史叙事,而不是硬凑的"东西方共鸣",且日本媒体会特别买账("起源于我们、传遍了世界")。

**四国里对我们最有利的是韩国**:韩国版比中日版多一个动作——**拇指盖章(도장)**。勾着小指的同时两个拇指对压,这叫"盖章",是韩国的基本动作(两个独立来源交叉验证)。而我们的三态是"收到→在办→**办妥(盖章)**",MODE_SPEC 里 B 模式 done 态就叫"盖章兑现"、视觉是印章。**韩国小孩拉完勾要盖拇指章——这不是我们要教给韩国用户的隐喻,是他们六岁就会的动作。** 在韩国,"拉勾=发起承诺 / 盖章=兑现承诺"这套语法是现成的、免费的、零教育成本的。

而且仪式的"惩罚条款"在每个文化里都存在且都很狠(日本吞千针 / 拳打一万下、中国"上吊"、美国"沉入地狱 / 针扎眼睛")——这解释了为什么这个符号配得上做"承诺协议"产品的内核:**它天生就不是"甜",它是"契约 + 违约罚则"。竞品都在卖甜,这个符号让我们可以卖"作数"。** 最值钱且唯一四国通用的资产是视觉——没有任何一家公司占住"两拳小指相扣、负空间是心"这个图形。名字要分四份,图形只需要一份。

## 7.2 首发日本:不是文化最贴合,是刚刚空出来

这是一个反直觉但有硬理由的选择。**文化最贴合的是韩国**(拉勾天生带盖章 + Between 信任真空);**但首发选日本,因为日本刚刚空出来**:

- **死亡名单**:Pairy(ペアリー)2026-04-08 停新注册、06-30 正式关服;Couples(カップルズ,Match Group 系,累计下载超 400 万)2020-08 关服。**日本情侣 App 不是没量,是没有 LTV——这不是"竞品弱",是"品类在日本被证伪过一次"。这直接决定日本的商业模式:钱必须在硬件上赚,App 订阅在日本是死路。**
- **当前 App Store 情侣类前三名全是韩国 App**(Between、SumOne、TheDayBefore),本土玩家全线退场;唯一本土 AI 情侣产品 COUPPLY(2025-06 上线)半年仅约 1 万注册 / 4000 对情侣,且它押的是 AI 情感陪伴(L3c 高危区,AI 调解吵架),不是 AI 执行闭环(L1+L2)——我们的切入点是"语音→任务卡→三态回执→AI 出方案"这条 execution loop + 硬件把它拽出屏幕。但必须抄 COUPPLY 一件事:E2EE(日本用户 + 情侣私密对话 + 中国团队,隐私是第一道生死线)。
- **有时效的窗口**:日本 2025-12 全面施行"智能手机新法",可合法绕开 30% IAP 抽成。加上 1.2 亿人口、全球第三大 App 消费市场。

**日本本地化关键**:① 文化上"察する(察言观色)不是障碍,管理才是"——日本社会自己正在劝女性"别指望被察觉,直接说出来",我们是在给日本社会已经开出的药方提供剂型;但"名もなき家事"里排第一的是"管理",所以蓝队解法是**AI 当 manager,不是她当 manager**(任务卡视觉发起者是系统不是她,回执是双向仪式不是单向验收,绝不做贡献度对比);② 界面别叫"任务(タスク)",叫**「おねがいごと(请求)」或「暮らしメモ(生活备忘)」**(MiniMax feedback-4:田中夫"リスト化されると業務報告っぽくない?");③ 日本 PR 核心口径直接用 SoftBank 官媒的话:"'考える・覚えておく'をAIに任せる"(把想着、记着、开口催这三件最重的无名家务交给 AI,她不用再当那个唠叨的人);④ 主战场是圣诞(情侣节)+ 交往纪念日,不是情人节;⑤ 硬件挂坠是日本真正的杀手锏(以心伝心文化 + 各在各的房间),建议找三丽鸥 / Miffy 联名;⑥ 数据必须落日本(APPI 第 28 条,跨境须在弹窗写明"中国");⑦ 只做 ASR 不做声纹。

## 7.3 韩国红线:셔틀앱 / 对称模式 / 数据不丢 / 声纹刑事

韩国机会与地雷同样锋利。机会:Between 信任真空 + 情侣市场回暖(2025 结婚数 +8.1%、珠宝 +13.5%)+ 74.9% 认同"恋爱也要讲效率"(연애효율화心智恰是任务 / 待办产品的天然叙事入口)。**会杀死你的三件事**:

1. **性别框架(셔틀앱 = 跑腿 App / 驯化男友 App)**:"她指派→他接收任务→回报已接收 / 进行中 / 已完成"在韩男社区眼里是셔틀앱,一旦截图挂上男性社区,24 小时内成 meme(GS25 "捏手指"判例:海报里一个手势被解读、全国不买运动、GS 股价当日 -2.37%、营销组长引咎去职)。**这不是措辞问题,是生死问题**——必须把"指派—验收"重构为"约定—共创"的对称设计(双向对称 UI、"우리의약속"共同事项、两角色对等主视觉);"派任务"叫"부탁(拜托)"不叫"지시(指示)";任务完成率对韩国男生是负担(和军队"점호(点名)"文化撞车),要弱化。韩国男人的精辟洞察(MiniMax feedback-4 민수):"한국 남자는 시키는 거 싫어하는 게 아니라, 시킨 게 티 나는 거 싫어합니다(韩国男人不是讨厌被使唤,讨厌的是被使唤这事儿被别人看见)"——所以必须无任何"战绩分享 / 情侣任务打卡"功能,默认"친족 차단(亲族屏蔽)",一切不可导出 / 截图 / 分享。
2. **数据事故(重演 Between = 出局 + 营收 3% 罚款)**:"数据永不丢失"是韩国市场生死线,PIPC 对"中国接收方"审查极严(阿里 ₩19.78 亿、Temu ₩13.69 亿已被罚),韩国用户数据必须不落中国、跨境同意条款写明目的地国名。
3. **声纹识别 = 刑事红线**:录语音→STT 转文字→任务卡(不做说话人识别)= 普通个人信息;但声纹识别 / speaker diarization = 민감정보(敏感信息),违反 = 5 年以下有期徒刑或 5 千万韩元罚金 + 全体营收 3% 过징금。情侣 App 有强烈"自动区分说话人"冲动,**第一版明确不做**——而我们是 1v1 结对产品,本来就不需要说话人分离,这是免费的合规红利。

其它:韩国必须内置 D-Day 引擎(100일 / 200일 / 300일 / 1주년自动生成 + 提前提醒,这不是加分项是入场券);"语音优先"在韩国是最大未验证假设(Between 早有语音 / 对讲机,韩国语音 AI 设备持有率仅 14.2%,上线前必须用户测试);军队场景(每年 20 万入伍男生 + 留守女方)+ 入伍前恋爱高峰是硬件挂坠的战略级机会(定价 ₩39,000–69,000 单只 / ₩69,000–98,000 对装,定位入伍 / 毕业礼物)。

## 7.4 美国:情感劳动地雷 + AI 是法理基础

美国原样上线必翻车。"她给他派任务 + 追踪 received / doing / done"在美国会被当作 emotional labor 的量化挨打(MIT Technology Review 专题、Reddit r/TwoXChromosomes 5,496 赞帖里"manager / boss"是骂人的词、"weaponized incompetence"是主流女性网络文化常备弹药,风险是"负向病毒传播")。**但反证里藏着正确答案**:Fair Play(Eve Rodsky,Reese Witherspoon 背书,有纪录片)证明"指派任务"不是原罪,能被接受的唯一原因是 CPE 三包(Conception 想到 + Planning 计划 + Execution 执行全包)——**所有失败的家务 App 都是让她保留了 C 和 P,只把 E 派出去,还多了一个 App 要维护。**

**美国战略判决:我们的 AI 自动出方案(L2)在美国不是锦上添花,它是整个产品的法理基础**——因为它是唯一能替女人承担 Conception + Planning 的东西。翻译成美国话:**"Fair Play, without the homework."**(Fair Play 的效果,但不用你自己发牌)或 **"Say it once. Never think about it again."** 五个可执行改造:AI 当 manager(任务卡发起者视觉上是 AI)、杀掉"assign"改"claim / own"("I've got this""Off your plate")、语音落地默认进"共享池"(unclaimed,任何一方可认领,物理删除"指派")、删除一切"已读了但没做"的可见性、绝不做性别化投放(统一叙事"No one should be the household's project manager")。美国合规硬雷:**BIPA(伊利诺伊)明文把声纹列为受保护生物识别标识符**,过失 $1,000/次、故意 $5,000/次(Otter.ai、微软 Teams 已被集体诉讼),分界线是"有没有从语音提取可识别身份的特征模板"——绝不做声纹登录 / 说话人识别是免费合规红利。美国竞品里 Milo(YC + OpenAI 直投,GPT-4 驱动,主打 invisible load)功能上与我们最像,必须持续盯。

# 8. AI 的角色与哲学

## 8.1 核心命题:两种生命点对点密度最高的沟通

把产品形态、功能、市场全部剥掉,只留最抽象的骨头,是创始人给的核心命题:**念念 EchoEcho,本质上是在服务"时空之中,两种生命点对点密度最高的沟通"。**

- **点对点(unicast)**:世界上的沟通拓扑无非三种——广播(朋友圈,一对多,接收方模糊可替换)、组播(群聊,多对多,责任稀释在人数里)、点对点(一对一,信号有确定的、唯一的、不可替换的接收方)。念念做的是且只是点对点,因为亲密关系的本质恰恰是"不可替换性"——他之所以是他,不是因为他能干活,是因为在你这里他不能被任何人替换。这就是为什么这个产品坚决不做开放社交、不做群聊、不做"情侣广场":每多一个观众,那条通道的私密浓度就掉一分。
- **密度最高**:亲密关系里的沟通是所有沟通形态里"每比特情感权重"最高的一种。同样一句"我记着呢",在工作群里是敷衍,在情侣之间是承诺——信息量一样,情感密度差了几个数量级。这也解释了为什么一个转写错误("给妈买"转成"给马买")在普通待办软件里只是笑话,在这里却是二次伤害:高密度通道里任何一点失真都会被情感权重放大。
- **为什么是情侣**:能同时满足"点对点 + 最高密度 + 长期高频 + 双向"四个条件的关系少得可怜(父母子女不对称且低频、挚友不承载日常琐事共同经营、同事可替换)。只有亲密伴侣把"最高情感密度"和"最高日常频次"叠在一起、还要长期共同经营一堆具体的事。从这里做对了才有资格往外扩(亲子、子女与父母),因为那些关系共享同一套底层——"非对称双人轻协作 + 情绪化反馈"。

## 8.2 距离感的辩证法:既保留等待,又打破隔阂

这是产品哲学里最微妙、最反直觉的一环,命题有矛盾的两半:一半是念念这种沟通天然带"距离感"(像古人写信,需要等待、无法即时兑现)——**在即时性泛滥的时代,延迟与等待反而制造珍贵**;另一半是用技术打破隔阂——无论两人身在世界哪个角落,想念的一瞬彼此就能感到对方的声音和震动(异国恋刚需)。这两半不矛盾,是同一件事的两个面:**我们打破的是"空间的隔绝",保留的是"时间的珍贵"。**

- **打破的是"物理隔绝",不是"距离本身"**:异地 / 军营 / 手术室的人最锋利的痛是"物理离线直接等于情感失联"。技术必须做的是让"看不到"和"不在乎"被彻底分开——三态里最轻的"收到"存在的全部意义就是这个,它是对"消失感"的解毒剂。硬件挂坠是这一半的终极实体化:想念的一瞬轻触胸口,对方那边亮一下、震一下,不显示内容、不打断,只是一次跨越八千公里的隔空击掌。
- **保留的是"等待",不是"迟钝"**:即时通讯用效率杀死了郑重("对方正在输入"让你连三秒沉默都焦虑,已读回执把"还没回"变成可指控的怠慢)。念念故意反着来,在通道里重新引入一点"古朴的延迟"——你说的话不是要求秒回的微信,而是一张任务卡(它承认"办成一件事需要时间",把"等待"制度化成"在办");系统在他进入心流 / 在手术 / 在开会时故意不打扰(延迟在这里不是 bug,是设计);而办妥那一下的分量,恰恰来自前面这段等待。书信之所以珍贵不是因为它慢,是因为它把"慢"变成了"郑重"。

一句话:最好的亲密,不是零距离,是"我随时能触达你,但我愿意为你等待"。零距离是黏腻、是监控、是即时性泛滥的另一个名字;而"能触达却愿等待"的距离,才是尊重、是从容、是爱里那份最难得的、给对方留出的余地。这也与韩炳哲《爱欲之死》的诊断严丝合缝:数字化、绩效化、透明化正在用"透明"和"秒回"杀死亲密里那个必须的距离,念念是对这个病的一剂微小但认真的解药。

## 8.3 把"记住"给机器,把"被记住"留给人

这是 AI 角色的一句话总纲,也是全部克制的浓缩:**AI 把"记住"的成本转移给机器,把"被记住"的感受留给人。** 它的锋利在后半句:AI 抢走的只能是成本(记住、整理、调研、提醒这些累人的活),绝不能抢走的是感受(被在乎、被记得、被用心对待的那份真实)。前半句是效率,后半句是红线。一个只做前半句的团队会做出"更聪明的待办 App"然后死掉;一个同时守住后半句的团队才可能做出"被用心对待的真实感"这门生意。

这条克制有它更深的历史与商业根据。2011–2017 年至少五家团队做过几乎一样的事(Pair/Couple、Avocado、Between、小恩爱、恋爱记),决定性证据是 Avocado——它 2012 年就做出"共享清单 + 共享日历 + 提醒",功能定义上就是一个没有 AI 的念念,团队来自 Google,媒体口碑好,仍然死了。这是一个干净的反事实实验:**当年缺的既不是需求也不是产品想法,缺的是把"她的话"变成"结构化任务"、把"任务"变成"可执行方案"的那个劳动力。** 没有这个劳动力,共享清单退化成"要两个人一起伺候的第二个记事本",然后被免费且已装机的微信 / iMessage 引力捕获。**历史的教训不是"现在时机到了所以稳了",是"现在及格线终于可以被够到了,而当年连及格线都不存在"。** LLM 第一次同时压掉了 C1(录入成本→趋近零)、C3(执行成本→减半),并中介化了 C2(协调成本——任务卡不是她建的,是 AI 从她的话里"长"出来的,"她派活给他"变成"AI 替他整理了她的心愿",指令经过第三方转译就不再是指令,AI 是关系里的缓冲垫)。

关于反命题必须正面回答:技术中介下的亲密是被增强还是被稀释?反方论点(从早期媒介理论到韩炳哲)有分量——任何技术中介都会稀释亲密,你让两个人隔着一个 App 相爱就像隔着玻璃拥抱。我们的回应分两层:第一层承认边界并用产品红线守住它(AI 只碰它碰了不会稀释的那部分=琐事的成本,坚决不碰它一碰就会稀释的那部分=心意的真实性);第二层反过来论证——对异地 / 军营 / 手术室的人,不是"技术中介 vs 直接在场"二选一,他们的直接在场已被物理现实剥夺,唯一选项是"技术中介 vs 彻底失联",对他们而言技术中介不是稀释,是**抢救**。分寸就是:对相守的人,技术要懂得退到隐身(海德格尔的"上手"),别打扰他们的直接在场;对相隔的人,技术要勇敢地成为那座唯一的桥。这个分寸,就是这个产品全部的手艺。

## 8.4 心理学地基:被看见、被回应、被记住

产品的每一条红线,都能被一根有名有姓的心理学柱子解释。以下引用均为确有其人其说,不确定处标"有一类研究认为",不编造文献。

- **依恋:成年人也在找那个"安全基地"。** 依恋理论由 Bowlby 提出,Ainsworth 用"陌生情境实验"落到可观测行为(婴儿把母亲当探索世界的"安全基地");1987 年 Hazan 和 Shaver 论证成年人的浪漫爱情本质上激活的是同一套依恋系统——你的伴侣就是你成年后的那个"安全基地"。依恋的安全感建立在"信号—回应"这个循环的可靠性上:一次可靠的回应加固一次安全,一次石沉大海动摇一次根基。念念在做的,就是把这个依恋循环做成一个看得见、不会掉的东西——她发出"念念"(信号),他"收到 / 在办 / 办妥"(回应)。这也是为什么"已读不回"杀伤力那么大:它不是一次沟通失败,是一次依恋系统层面的"安全基地消失"警报。
- **回应性:亲密关系满意度的头号变量。** Reis 和 Shaver 的"亲密作为人际过程"模型中,被反复验证为满意度核心预测因子的是"感知到的伴侣回应性(perceived partner responsiveness)"——决定一段关系幸不幸福的往往不是"你为我做了多少事",而是"我觉得你有多懂我、多在乎我"。注意那个"感知"——重点不在客观付出,在主观被回应的感觉。EFT 创始人 Sue Johnson 把它讲成 A.R.E.(Accessible 可触及、Responsive 有回应、Engaged 投入),伴侣间最深的那个问题永远是"当我需要你的时候,你在吗?"。这根柱子解释了每一条红线:为什么三态里最轻的"收到"如此重要(它是最低成本的"我在,我看到了");为什么绝不做"已读时长"(回应性的关键是"感知到的在乎"而不是"被监控的证据",一旦她能看到"他已读未回六小时",产品交付的就不是回应性,是监控焦虑,它会摧毁自己本想创造的东西)。
- **沉默为什么最伤:被无视是一种社会性的痛。** 社会心理学家 Kipling Williams 用几十年研究"社会排斥",尤其是"沉默对待"。著名的"网络掷球(Cyberball)"实验发现,哪怕只是被两个陌生电脑程序在虚拟传球游戏里冷落,人都会感到真实痛苦;脑成像研究(如 Eisenberger 等)甚至观察到社会排斥激活的脑区与生理疼痛高度重叠——被无视在神经层面真的"疼"。这解释了一个反常识现象:很多来吐槽的用户气的不是对方办砸了,是对方"没反应"。晓萱要是收到"床头灯我知道了,但这个月手头紧晚点补给你行不行",她不会截图发闺蜜;让她受伤的是那个红包背后的沉默。所以产品的第一使命,是让"沉默"这个最伤人的东西在这条通道里消失——这是把"收到"设为最低门槛动作的深层原因。
- **无数个微小的"转向":关系是在日常里一点点攒出来的。** John Gottman 提出"连接的邀约(bids for connection)"——亲密关系不是靠几次盛大浪漫维系的,是靠日常里无数微小邀约来回攒的:她说"你看那只鸟",表面说鸟,底下在说"陪我看一眼、跟我连一下"。对方有三种回应:转向(搭理她)、转开(忽略)、转身对抗(怼回去)。婚姻幸福的"大师"日常里"转向"的比例远高于走向离婚的"灾难",差距大到可以据此预测走向;他还有个著名的"5:1"观察(稳定关系里正负互动比例约五比一以上)。这把念念的战场从"记住一件大事"降维到它真正的战场——日常里成百上千个微小的、随口的连接邀约。而残酷的现实是:大量 bid 不是被故意"转开"的,是被生活噪音(工作群、99+、心流、酒局)意外地漏接了——善意在,能力不在。念念做的精确地说就是把"漏接的 bid"变成"接住的 bid",三态里的"收到"就是被产品化了的、永不漏接的"turning toward"。这也直接支撑了修勾模式"每一次怼必须有一次和好收尾"的设计约束。
- **承诺与仪式:为什么"作数"比"甜"更耐久。** Caryl Rusbult 的"投资模型"说承诺由三个变量决定:满意度、替代选择的质量、以及投资(你在这段关系里沉淀了多少无法带走的东西)。投资越厚,承诺越牢,越舍不得走。竞品都在卖"满意度"里最轻的那一档("甜"——每日一问、情侣游戏、撒糖),但"甜"是最容易被替代、最容易在热恋期后蒸发的东西;念念卖的是"投资"——两个人共同办成的一件件事沉淀成一本"着落档案"(她说过 214 件事,他办成了 187 件),删掉这个 App 等于删掉两个人的一段共同历史。这就是为什么"办事"比"撒糖"更耐久,也是"数据永不丢失"原则的心理学根。而仪式感是投资的加速器——有一类关系研究认为共享的仪式(纪念日、固定的小约定、承诺的手势)能显著增强承诺感与身份认同,因为仪式把抽象的"我们"变成具体的、可重复的、有形的动作。这正是把"拉勾"这个跨文化承诺仪式做进产品的原因:拉勾=立约,盖章=兑现。

## 8.5 社会学地基:原子化时代里一个逆流而上的封闭单元

把镜头从两个人拉远到整个社会,念念踩中的是几条同时发生的结构性趋势。

- **原子化:亲密关系正在变成稀缺品。** Robert Putnam 在《独自打保龄(Bowling Alone)》里描述了现代社会"社会资本"系统性流失——人们参与的社群、稳定的连接、面对面的往来都在减少,人越来越像悬浮在城市里的原子。项飙用更中国、更锋利的说法命中同一件事:"附近的消失"。在这样一个世界里,一个稳定的、点对点的、不可替换的亲密连接既稀缺又珍贵——当社群在瓦解、附近在消失、朋友越来越像微信列表里不再联系的名字,那个"对我来说不可替换的一个人",成了很多人对抗原子化的最后一个据点。我们不是在争夺一块正在缩小的蛋糕,是在服务一个因为周围一切都在瓦解而变得越来越重要的堡垒。
- **韩炳哲的警告:数字时代他者正在消失。** 韩炳哲在《爱欲之死》《透明社会》里给出更尖锐的诊断:在数字化、绩效化、透明化的今天,真正的"他者"正在消失,我们被困在"同质化的地狱"里,social media 让我们只照见自己的倒影;而爱欲的前提——那个不可被我完全掌控、保有神秘和距离的"他者"——正在被"透明"和"即时"消解。这和距离感辩证法严丝合缝,并给了它社会学的重量:即时通讯和开放社交正在用"透明"和"秒回"杀死亲密里那个必须的距离,念念反其道而行,在一片要求"全透明、秒回、随时可查岗"的数字荒漠里,为两个人重新围出一小块有距离、有等待、有神秘的自留地。
- **低结婚率但恋爱没减少;异地在结构性增长。** 有人会问结婚率这么低做情侣产品还有市场吗?数据给的答案是反直觉的:以韩国为例,2025 年结婚数 24 万件同比 +8.1%(回声潮世代进入婚育),出生数连增 10 个月;更关键的是 74.9% 的韩国人认同"恋爱也要讲效率"。低结婚率不等于低恋爱率——人们可能更晚、更谨慎结婚,但恋爱这件事一点没少。而且现代人的恋爱结构本身在变:求学、工作、外派让异地恋、异国恋成为一种结构性常态而非例外,念念对"物理隔绝"的解法踩中的正是这个正在变大的结构性缺口。
- **同一个产品在不同关系文化里,可能是解药也可能是地雷。** 这是全球化调研里最重要也最惊险的发现。在日本,社会正在劝女性"别指望被察觉,直接说出来",我们不是在挑战日本文化,是在给日本社会已经开出的药方提供剂型;但日本有另一颗雷("名もなき家事"里排第一的是"管理"),一个"她说话→生成任务卡→她盯他的完成状态"的产品可能被读成"又给我加了一份管理工作"——蓝队解法是让 AI 当 manager 不是她当 manager。在韩国,机会("恋爱效率化"心智 + Between 信任真空)与地雷(性别对立 + GS25 判例)同样锋利。这个东西是这个产品被逼着长出来的最深一层智慧:它处理的是人性,而人性在不同社会土壤里长出的敏感点不一样——三条红线既是产品宪法的来源,也在日本、韩国两个市场被独立地再验证了一遍。

## 8.6 终局:从一句话,到一层"关系的操作系统"

一个产品十年后是什么样,决定了今天该怎么做它。念念的软件做的是"把说的话变成办成的事",但软件有天花板——它活在屏幕里,而屏幕是需要你主动去看的。硬件挂坠(EchoCharm)要解决的就是"屏幕的缺席",它的产品定义不是"一个能提醒的蓝牙配件",而是"想念的实体开关":她记下一条心愿,他胸前的挂坠轻轻亮一下、震一下(不显示内容、不打断当下,只是"有人在惦记你"的一次微光);在最纯粹的形态里它甚至不需要一条具体消息——想念的那一瞬轻触胸口,对方就感到你的声音和震动。硬件卖的不是提醒功能,是"信物感",也是商业上对抗"热恋期后流失"的重武器(一个挂坠是关系的"投资凭证",删 App 容易,取下一个天天戴着的信物难)。

把时间拉到十年:如果这条"说一句→有着落"的最短的路真的成了两个人日常的默认通道,那么念念沉淀下来的就不只是一堆任务记录,而是一段关系全部的"共同记忆资产"——越用越懂这段关系,迁移成本越来越高,这是数据资产的复利,也是与一切通用工具的终极分野。它会自然地跟着关系的生命周期迁移(从拉勾勾到念想,从两个人到一家人),甚至横向长出新的关系单元(子女与父母)。所以十年后的念念想成为的,是一层薄薄的、隐身的、覆盖在所有"最亲密的一对一关系"之上的操作系统——它不制造关系,只做一件事:让"我在乎你"这件看不见的事,永远有一条不会漏接的通道。

# 9. 技术路线

## 9.1 iOS 原生 SwiftUI:这次不是品味偏好,是技术事实

念念要的六件事(spring 物理动效、Haptics 微震动、录音波纹、办妥仪式、锁屏 / 灵动岛提醒、Live Activity)里,**灵动岛 / Live Activity / 锁屏组件这一类,四条技术路线(SwiftUI / React Native / Flutter / Capacitor)全都要求你回到 SwiftUI 写 Widget Extension**——跨端框架在这里省不掉原生工作。而"跨端"唯一的收益(iOS+Android 共享代码)因为**安卓已暂缓而归零**。选跨端框架你并没有"避开 Swift",你只是"既要维护 JS/Dart 主工程,又要维护一个 Swift 副工程,还要在两者之间桥接数据"——复杂度不降反升。Capacitor 能吃到 70% 代码复用,但它复用的恰恰是"该被重做的那部分"(念念转原生的动机本身就是"网页的跟手感不够",用 Capacitor 把现有网页装进 WKWebView 等于把想解决的问题原样带进去)。**结论:SwiftUI 原生,唯一推荐;Capacitor 只作"上架占位 / A-B 验证"的临时手段,不作终局。**

## 9.2 技术栈与后端复用

- **前端**:SwiftUI(iOS 17 为基线,兼容 16)+ ActivityKit / WidgetKit(灵动岛 / 锁屏组件)+ Core Haptics(自定义 AHAP 波形,唯原生可做)+ AVFoundation / AVAudioEngine(取实时 PCM 电平驱动"暖珊瑚双圈波纹")+ SwiftData(本地副本,呼应"数据永不丢失")+ 仅在"办妥仪式"一处用 Rive(复杂矢量动效,主界面所有微交互用 SwiftUI 原生,不为炫技引入运行时)。
- **后端 100% 复用现有 Cloudflare 栈**:`functions/api/[[path]].js` 已是标准 REST + Bearer token 鉴权,原生 App 用 URLSession 携带 token 调用即可,**后端一行不改就能服务原生客户端,零迁移**。已有端点覆盖结对 / 消息流 / 音频 / AI 僚机方案页 / 硬件通道。只需补两块:**APNs 推送**(替换现有 Web 通知 + 12s 轮询,Cloudflare Worker 用 Web Crypto 签 ES256 JWT,APNs 本身免费)和**真账号体系**(替换现有免注册匿名 token)。
- **账号体系**:App Store 4.8 只有在提供第三方社交登录(微信 / Google)时才强制 Sign in with Apple。首发推荐"手机号(自有)+ Sign in with Apple"双通道(换机不丢、隐私友好、审核最稳、日韩接受度高);中国区上架时加微信(但一旦加第三方登录必须同时保留 SIWA)。现有邀请码结对逻辑完整保留,只是把"匿名 member token"升级为"account 之下的 member",一个 account 可属多个 space(天然支持多小家 / 一对多结对)。
- **域名**:需正式自有域名 + Associated Domains 才能用 Universal Links(邀请码 deeplink:点 `https://域名/join/NN-XXXX` 直接拉起 App 到加入页,未装则落地页引导下载)。

## 9.3 开发方式:Claude Code 主 + 设计工具辅

`MOTION_DESIGN_PATTERNS.md` 已把动效沉淀成 Motion Tokens 规格书(3 条 easing、4 档 duration、stagger、位移全定死),这本身就是一份可直接翻译成 SwiftUI 的实现说明书——Claude Code 把它落成 `Animation` 常量与 `ViewModifier` 是机械翻译,不是再创作;结构性工作(网络层、SwiftData 模型、APNs、状态机、五模式主题系统)是 Claude Code 的强项。**唯一铺张时刻「办妥→回响仪式」**先用 HTML / Figma 做视觉探索定稿(3 个变体真实预览),再由 Claude Code 落 SwiftUI 并在真机反复调曲线。别把整个工程都塞给设计工具(维护性差),也别指望纯文字规格喂出有灵魂的仪式动画。

## 9.4 schema 演进(需老板拍板,红线操作)

原则(PRD §5.3 + 全局宪法):优先"仅加列 / 加表",绝不破坏现有数据;加表 / 迁移属红线操作须用户拍板。iOS 文档 §9 建议方案(全部向后兼容,`ALTER ... ADD COLUMN` + `CREATE TABLE IF NOT EXISTS`,现有网页 / 硬件客户端零感知):新增 3 表(`ee_accounts` 真账号叠加在 member 之上不替换、`ee_push_tokens` 推送 token、`ee_anniversaries` 纪念日 / 周期念念);`ee_members` 加列(account_id 可空 / 勿扰起止 / settings JSON);`ee_items` 加列(priority / tag / deadline / remind_plan / reply_to / recur_id / deleted_at 软删除 / enc 加密预留 / research_key)。唯一建议在原生化同期就做的减负动作:把 `research_html`(方案页 HTML 可能几十 KB/条,是 D1 10GB 硬顶的唯一隐患)移出 D1 存 R2。

## 9.5 到 TestFlight 的里程碑

单人 + Claude Code 协作粗估(工期均为估算):M0 立项(2–4 天)→ M1 网络层 + 数据模型(3–5 天)→ M2 核心界面三态(5–8 天)→ M3 账号 + 结对 + 免注册迁移(5–8 天,R1 最高风险)→ M4 推送 APNs(4–6 天)→ M5 办妥仪式 + AI 僚机(4–6 天)→ M6 五模式主题(3–5 天)→ M7 Live Activity + Widget(4–7 天)→ M8 打磨 + 合规 + 冒烟(4–6 天)→ M9 TestFlight(2–5 天 + 审核)。**合计约 6–9 周达 TestFlight,MVP 版可压到约 4–5 周**(M0–M5 + 精简 M8,把五模式 / Live Activity 放到 TF 后迭代)。最大风险按严重度:免注册→账号迁移丢数据(生死级,踩韩国"数据永不丢失"生死线,对策:account 叠加不替换、旧 token 永久有效、迁移前后本地副本双保险、灰度、可回滚)、中国大陆 ICP 备案(需大陆法人主体,首发走海外绕开)、Cloudflare 单点 + D1 10GB 硬顶、声纹刑事红线(第一版起就不做)。

## 9.6 UX 走查:六类用户顶出的交互红线

UX 走查(Kimi 生成,六类用户视角:手残党男生 / 细节控女生 / 50 岁阿姨 / 程序员 / 异地恋 / 新用户)顶出一批必须在原生重做时守住的交互底线,按影响面排序的 Top10 里最关键的几条:

- **三态按钮易误触、状态变更无二次确认**(手残党 / 阿姨 / 异地恋):任务卡"在办→办妥"必须加底部确认面板 / 二次确认,按钮尺寸≥48×48px,点击后 disabled + loading,状态色区分蓝(收到)/ 橙(在办)/ 绿(办妥)——现有三态按钮"一串小按钮挨着,一点就误标成办妥"是高频痛点。
- **首次打开无 onboarding、空状态冷清**(新用户):开屏加 3 步引导轮播(创建小家→给 TA 发一件小事→用邀请码让 TA 加入),主界面空状态显示半透明"示例任务卡"("给 TA 发一句:记得喝牛奶")+ 输入条黄色描边高亮提示——三个流失点全部集中在"不知道这是干嘛的、担心没账号会丢数据、发出第一条后不知如何完成闭环"。
- **语音发送流程不可见、失败无反馈**:录音后显示"上传 / 转写 / 发送"进度条,失败给"重转文字"或"直接发语音"选项——语音是产品第一入口,任一环节失败用户无感知是致命的。
- **AI 方案直接渲染完整 HTML,有 XSS / prompt injection 风险**(程序员):方案内容改 Markdown 渲染,必须用 HTML 时先 DOMPurify 消毒再置于 iframe sandbox 禁 script——这条是安全红线。
- **无账号体系、token 本地易丢**(阿姨 / 新用户):创建后提示"数据已保存在本设备,建议绑定手机号或复制邀请码",引入 access/refresh token——直接对应"数据永不丢失"原则与账号迁移的最高风险。
- **办妥情感反馈只有一方、发起方通知弱**(异地恋 / 阿姨):执行方点"办妥"后双方全屏庆祝动效 + 发起方手机推送 + Toast"TA 已完成 xxx"——异地恋最想要"一起庆祝"的感觉,这是回响仪式的双端强化。
- **术语看不懂**(阿姨):"小家""回响""AI 僚机"都是年轻人的词,首次出现时旁边给黄色问号图标点击弹白话解释("回响 = 对方看到并回复你")——这条直接印证了"爸妈 / 家人版"需要完全不同的产品语言(把"任务卡 / 三态流转 / 进度百分比"这些工业词汇换成"今天怎么样 / 我很好 / 记得啦")。

这些交互红线不是锦上添花,它们决定了男方(手残党代表)敢不敢用、阿姨(适老代表)用不用得起来、程序员(安全代表)信不信任——每一类都对应产品的一个生死变量。

## 10.4 二十四个 Use Case 的产品机制缩影

24 个 Use Case(6 类:日常琐事 / 纪念日礼物 / 大宗决策 / 异地时差 / 多元关系 / 硬件场景)反推出的 Top10 功能排序,是产品优先级的原始依据。这里用几个最能代表产品灵魂的场景,把"说一句→有着落"的机制讲活。

- **UC-01 红糖姜茶**:林小满姨妈第一天窝在宿舍给读研男友发语音"我想喝红糖姜茶"。没有产品时他回一句"好的"就沉进实验,晚上想起来奶茶店已打烊——那两个字"好的"根本没进脑子,她收获的不是姜茶,是"他对我就这么点上心"的闷气。有产品:她按住说一句→系统转成任务卡并自动识别"奶茶店打烊前"这个隐含 deadline,AI 僚机附上沿途 3 家还在营业的店→他点收到 / 在办 / 办妥。**回响结果**:她收到"在办"的那一刻就已经不委屈了——她要的从来不是姜茶,是"我的话有人接住了"。这一句是整个产品情感逻辑的原点。
- **UC-07 生日红包之外**:赵晓萱生日前两周在语音里认真说"我想要小米那款智能床头灯",男友在深圳忙成陀螺。没有产品时她只收到一个红包和"生日快乐"四个字,一个人截图发闺蜜吐槽——异地的"被遗忘感"会被灾难化成"我们是不是越来越远了"。有产品:那句话被转写成心愿卡静静躺在他的"她提过的事"清单里→生日前 10 天系统提醒"该下单了,快递到北京要 3 天",AI 僚机给出比价链接→他一键下单,物流自动同步进度%。**她拆开的不只是灯,是"两周前那句话他真的记到了今天"**。这是 BP_MANIFESTO 序章那条"没有着落的消息"的正解版。
- **UC-04 ADHD 丈夫的外部大脑**:大鹏(38 岁,自述 ADHD 倾向)被老婆连续三天提醒换灯泡,嘴上答应大脑五分钟后自动清空,冰箱贴看惯了就视觉过滤。AI 僚机替他把笼统指令拆成可执行小步骤(周五晚下单灯泡附链接,周六 9 点出门提醒 + 导航),每完成一步进度%像游戏经验条往前走,办妥弹一句明确的"你做到了"。对他这样的大脑,这不是备忘录,是救命稻草式的外部结构化系统——而"办妥"的正反馈是他婚姻里稀缺了十年的成就感。这个 persona 是"给男方设计 Aha 时刻"的活样本。
- **UC-17 反监控专项 · 住院医的欠账清单**:周则明一周两次 30 小时连班,女友的猫疫苗、下夜班带稀饭、生日晚餐全在他"半宕机"状态里蒸发。她的话都进任务池,AI 结合他的排班把提醒自动挪到查房间隙、下夜班后这类"合适的时间",忙时绝不推送;他下班收到"欠账清单 + 补救方案"。**反监控设计**:她那边永远看不到"他已读未办 6 小时"这种审判性数据,只看到任务在他下班后被逐一点亮——系统替他挡住了"你连这点小事都办不好"的量化羞辱,也替她省掉了反复追问的委屈。UC-03(P0 故障夜)、UC-12(方案页署名"AI 僚机为你准备"不是"她布置的作业")、UC-20(酒局丈夫的晨间清单,把"她催他"这个动作彻底移除)三个专项共同构成"反监控"原则的产品化。
- **UC-16 军营的一个月**:阿兵手机上交约一个月一次通话窗口,上线面对几百条消息分不清哪句是随口说说、哪句是真急事。AI 僚机去重、分级、合并成"3 件事:①婚期要请假(急)②她病了想听你说话(情绪)③生日想要花(有 deadline)",他窗口期 10 分钟处理完。**三态的存在本身就是对"消失感"的解毒剂**——她终于分得清"不理我"和"看不到"的区别。这是 F-12 消息队列化 + AI 去重分级的原型,也是"AI 是零成本录入时代的中央银行"的实战版。

Top10 功能排序(按 24 个 UC 出现频次 × 命中调研共性 × 守住情绪红线):语音转写自动建卡(P0 地基)> 三态状态流(P0 协议层)> AI 僚机出方案(P0 唯一付费锚点)> 场景感知智能提醒时机(P0,提醒与骚扰的分界线)> 反监控信息隔离(P0,决定执行方留存的负向功能)> 进度%可视化(P1,"看"替代"催")> 硬 deadline 递进提醒(P1)> 办妥正反馈回路(P1,男方留存钩子)> 消息队列化去重(P2)> 硬件免手持入口(P2)。

## 10.5 三十个新场景的三条贯穿洞察

场景脑暴 30 个(节日 / 生活转折 / 微妙关系 / 群体 / 挂坠专属各 6 个)给出三条可直接进 PRD 的洞察:① **"代办"是新型表白**——30 个场景里有 11 个本质上是"用做代替说"(不会说出口的道歉→"帮你把脏衣服洗了 / 明早闹钟帮你定好",对方收到的是被照顾不是"对不起"三个字),这是念响最深的护城河,任何竞品都难抄;② **挂坠的本质是"在错位的时空里喊一声"**——开车 / 做饭 / 哺乳 / 遛狗 / 远在海外,双手被占、视线被占、注意力被占但脑子是空的,挂坠是那些时刻你的"第二张嘴",5 个时空错位场景强烈建议首发挂坠 demo 全覆盖;③ **"高情感低频"才是新蓝海**——临终告别、冷战破冰、道歉、宠物临终,这些场景频率低但情感浓度高、口碑传播力极强,可作品牌故事素材库而非主营收功能(没有任何市面产品做"临终仪式"这个高情感浓度场景)。

**明确的功能红线(舰队红队顶出,写进设计禁区)**:砍掉 / 淡化"进度 %"的量化压迫(改二元 + 情绪标签"在办 / 在忙 / 有点累"是可选方向);卡片禁止"超时 / 逾期"字眼(只允许"已办妥 / 不需要了 / 改天再说");24h 自动溶解机制(过夜的鸡毛蒜皮自动消散,避免怨气沉淀成档案);分手后关系状态可一键切"已分手",双方独立确认后 72h 内双向抹除原始数据(只留极简摘要);硬件只走近场通讯、无 GPS / 无麦克风常开 / 按住才录(切断"位置追踪"与"监听"想象)。

## 11.0 前置事实:主品牌名必须重选

`docs/PRD.md` 2026-07-12 记录"主品牌暂定「念想」"。但命名调研已确认:**「念想 / 念响」已被注册**(念念手帐、香柚信息情感记录同赛道、Reecho 睿声 AI 语音克隆)。"念 / 响 / 回 / 音"四个字里,响、回、音三个字的独占空间基本被吃光了。所以主品牌名必须往"口语短句"和"念 / 想 / 应 / 着落"偏移——那里还有空地。这是三条路线出现的直接原因。

## 11.1 路线一:拉勾系(出海优先,概念名实合一)

**代表候选**:「我们拉勾」(主推)/「一百年不许变」/「不许变」;全球拉丁文字标 `Pinkypact`(备选 `WePinky`)。

**优势**:① 出海最强——拉勾是中日韩美同源的品牌内核,零解释成本讲清"这个 App 是干什么的";② 承诺 + 兑现的语法直接映射三态(勾=立约、章=兑现);③ "我们拉勾了,那就算数"是一句不需要 App 存在也成立的情话,"我们"两个字把它锁死在双人关系里(任何 To B / 办公场景用不了这个名字);④ 童谣的惩罚条款让它天生卖"作数"而非"甜"。

**商标 / ASO 现状(实测)**:「拉勾勾」两项全中(ASO 被招聘 + 货运双重寄生、"拉勾"是拉勾网在先知名商标)——加语气词(啦 / 吧 / 了 / 个)救不了,唯一钥匙是**加人称代词**。「我们拉勾」App Store cn/us `resultCount = 0`(41 个实测词里唯一"含拉勾却零结果"的),显著性从"大概率被驳回"跨越到"大概率能下证"。童谣原句截取是最大免费午餐:「一百年不许变」「不许变」「拉钩上吊」cn/us 全部 0(与"拉勾"零字面重合,风险源不存在)。

**纠结点(必须如实摆出)**:① **幼齿风险**——"儿童仪式"对 25+ 已婚人群(老K 32 岁 / 大鹏 38 岁)可能觉得幼稚(缓解:这正是五人格模式的价值,拉勾勾是年轻派皮肤不是唯一皮肤,品牌内核是"承诺 + 兑现",表皮可以是童谣或印章契约);② **仪式认知度高 ≠ 转化率高**——它降低的是"这个 App 是干什么的"解释成本,不解决"我为什么要装"(不要把品牌内核当增长引擎用);③ **出海名字撞墙**——日本 YUBIKIRI 已被注册且命中交友类目(GTM 默认假设被推翻,日本要另挑名),美国 Pinky Promise / Swear 三重死锁(儿童癌症慈善 + App 拥挤 + Littlefinger 背叛者语义);④ **容器句拗口**——"我记在「我们拉勾」里了"别扭,但它在动作句"这事我们拉勾了"里发光,这笔账划算;⑤ 域名 `pinkypact.com` 已被占(2023 闲置域名),可用 `.io` / `.co` 或谈收购。

## 11.2 路线二:接住系(与「」logo 名实合一)

**代表候选**:「接着呢」/「我接着呢」/「兜住」;英文造词 `Heldly` / `Held`(logo 文件里 07 号方向英文代号本就叫 HELD)。

**优势**:① **名实合一是本路线最大战略价值**——回声括号「」上半是她说出口的引号、下半是他接住的那一半、中间空白就是"话没掉在地上",这个符号本身就是"接住"的图形。一个叫「Reecho」的 App 配「」会被读成取景框;一个叫「我接着呢」的 App 配「」只可能被读成"一句话被两只手接住",名字给符号上了锁;② "接"字一字五关全部命中产品(接话 / 接事 / 接人 / 接住 / 接通),语义密度比"勾"更高(勾是器物,接是动作,而动作能被情侣说出口);③ 心理学母体扎实——"被接住"在依恋 / 婚恋讨论里是成型术语(我倒出来的东西有人在下面接着,没掉在地上);④ 三态天然成句("我接着呢 / 我办着呢 / 我办完了"全部第一人称、全是他会说的话,这不是话术设计是人格设计)。

**商标现状(实测)**:「接着呢」「我接着呢」「有我接着」「都接着呢」App Store cn `resultCount = 0`(口语短句 = 天然的商标空地);「兜住」cn 无同名 App(返回"一兜糖 / 竹兜育儿"同音不同名)。共同风险:日常口语作商标可能被 CNIPA 以"缺乏显著特征"驳回——对策是中文字标做强设计 + 图形组合申请 + 英文造词组合注册(可控技术问题,不是死局)。

**纠结点**:① **太软**——"接着呢""我接着呢"在应用商店的图标矩阵里不"跳",需视觉端补力量,而且英文没有好对位(`Got It` 太平);② **"我接着呢"是一句话不是一个词**——四字图标下面显得"不像个正经产品"(但这可能恰恰是护城河:所有正经产品都记不住,只有人话记得住);③ **"兜住"的取舍**——「兜住」同时是动词 + 器物(能做成挂坠里真的小兜,硬件叙事最强,且"兜"的兜形弧从图形层解决了「」纯几何无曲线的先天缺陷),但英文 `Doudou` 已被"Baby Tracker for Two"占(相邻品类),且"兜"字有点土;④ **"接住"两字本身**被"接住 XX"的小游戏命名范式拖累 ASO。

## 11.3 路线三:念想 / 回应变体

**代表候选**:「惦记」/「记心上」/「有我呢」/「有着落」/「声声应」/「小惦记」。

**优势**:① **保住原品牌血脉与语感**("念念不忘、必有回响""事事有回应")而避开被占用的确切词;② 体温高——「惦记」是本轮体温最高的双字词(中文里从来没有人惦记一个 KPI,天然只用于人对人),「记心上」是唯一"名字 = 完整情话"的候选("我记心上了"是中文里男人能说出的最重的承诺之一且不油腻),「有我呢」是体温最高的名字(三个字给的是"我在"的最大安全感,且"收到"态直接就是品牌名,品牌曝光与产品使用完全同频);③ 定位清晰——「有着落」不卖甜卖"你可以依赖这个人"(对 22 岁不 sexy,对 32 岁伴侣才致命,是选定更年长更愿付费用户群的商业判断)。

**商标现状(实测)**:「有我呢」「记着呢」「想着呢」App Store 未检索到同名(日常口语显著性风险,对策同接住系);「惦记」App Store 未发现同名 App 但高频通用词(9/42 类大概率已有他人商标,防御打法:"惦记 App"组合注册 + 图形 + 主推"不用惦记了"独占口号);「小惦记」是「惦记」的商标保险版(三字组合显著性更高);「记心上」绕开了"心上"(二手奢侈品平台)但与之在 9 类可能被引证;「有回音 / 声声应」落在"回音—语音社交"极度拥挤赛道(回音 / 小回音 / 回音岛 / 腾讯回音交友四家占领,主名不可用)。

**纠结点**:① 多为好 slogan 被误当名字("都接住""放心说""随便"等大量候选是好 slogan 弱名字,好点子不必和好名字绑定);② 与「」logo 的名实合一断裂(选"记 / 想 / 应"系等于弱化接住系最大战略资产);③ 显著性与口语性天然对立(越干净的词越没人说,越有人说的词越注不下来——「想头」商标最干净体温最勉强是这个张力的标本)。

## 11.5 本文的倾向(不替老板拍板)

**国内主品牌**:倾向接住系("我接着呢 / 接着呢")或「记心上」——理由是名实合一(与既有「」logo 资产咬合)+ 体温高 + 男性敢用(男性安装率是这个双边产品的生死线,视觉必须反甜腻)+ 商标空地。**出海**:倾向统一走"拉勾图形 + 产品语法(勾 / 章)+ 四国本地名"——图形是唯一四国通用、无人占用的最值钱资产,名字各国各挑(中国"我们拉勾"、韩国"손가락약속 / 꼭꼭약속"、日本另挑干净词、美国 Pinky Pact 或纯图形)。**这意味着国内名与出海内核可以分层**:国内用高体温的接住 / 惦记系做主品牌,出海用拉勾图形做全球符号——两者靠"承诺 + 兑现"的产品语法统一,不冲突。但这是老板的品牌品味决策,本文只摆牌不替拍。

- **拉勾系死路**:拉勾勾(招聘 + 货运双寄生)、拉勾啦 / 拉个勾(货运)、小拇指(浙江著名商标 3713 家汽修门店 + 465 条商标防御,比拉勾更死)、勾一下(美区返回一个 ToDo List,机器诊断它就是工具)、拉钩上吊(注册净土但"上吊"触发自杀关键词审核,上架雷区)、尾指之约("之约"后缀 100% 二次元手游)。

## 11.7 视觉资产:一个 logo,三层阅读

命名的纠结之外,视觉是唯一四国通用、无人占用的最值钱资产,必须单独讲清。老板此前对现有 logo 的痛点是"看不出小指勾小指"——原因是解剖学的:一根手指离开手掌就不是手指了,只是一条线,人眼识别"小指"靠的是旁边那四根蜷起来的手指(小指之所以是小指,是因为它是那个唯一伸出来的)。修复方案(全部候选共用的视觉总纲):① **必须画拳头**——两个圆润握拳各自只伸出小指、侧视、两根小指相扣,一秒就能读懂;② **用负空间藏心**——两根小指勾在一起时中间围出的空隙天然是一个心形,把这块负空间填成品牌色,一眼是手、再看是心,这是这个母题唯一的、别人还没用掉的视觉金矿;③ **勾扣处 = ✓ 的转折点**——两指相扣的折角和打勾 ✓ 的折角是同一个形状,缩到 16px 时退化成一个 ✓ 依然成立。**一个 logo 三层阅读:拳头小指(0.2 秒)→ 负空间的心(1 秒)→ 缩小后是个 ✓(功能层)。**

如果走接住系,还有另一套同样成立的符号:回声括号「」——上半是她说出口的引号(墨青),下半是他接住的那一半(珊瑚),中间那块空白就是"话没掉在地上"。它原本的风险(纯几何、无曲线、"温柔"全靠字体色彩补)恰好被"兜住"这个名字解药——下引号从直角折线微调成一个极浅的兜形弧,就从图形层解决了先天缺陷。两套符号对应两条命名路线,老板选名字的同时也就选了视觉母题:拉勾系=两拳小指相扣负空间藏心,接住系=回声括号「」。

硬件信物与视觉母题是一体的:拉勾系天然长出"一对不对称、可磁吸相扣的挂坠"(她一瓣他一瓣,合起来才是完整 logo,单卖没有意义,天然双份客单价);"一百年不许变"能长出"一枚真的能蘸印泥、盖在纸上的微型印章挂坠"(可脱离 App 单独卖、毛利高、是能送人的信物);接住系"兜住"能长出"一枚小小的金属兜(可开合),里面塞一张手写纸条"(唯一"挂坠本身有功能"的候选)。这三种硬件形态都不是"会震动的塑料片",而是"能攥在手心、贴在胸口的信物"——把"在场"这件抽象的事做成一个物,这是第四章"打破隔阂"命题的终极实体化,也是软件平台的射程盲区。

# 13. 风险总账

每条风险给出来源、严重度与缓解措施。这些风险大多是"可以被产品设计正面进攻"的,而不是结构性死局——这是它和"没有 LTV、纯靠甜维系"那类产品的根本区别。

## 13.1 产品风险

- **爱意工具化 / KPI 化(最尖锐的一问)**:把爱情装进有"任务卡 / 进度% / 办妥率"的系统,会不会让两个人用管理项目的方式管理感情?**缓解**:这个风险是真的,而它正是全部产品克制的来源——五条第一性原则(尤其去 KPI、反监控)+ 三条一票否决红线就是对抗工具化的宪法。更深一层:我们要工具化的恰恰是"为了去工具化"——把记住 / 整理 / 催办 / 调研这些会把人变成"婚姻里的项目经理"的事务性劳动交给系统,解放的是心意的空间。舰队红队的具体建议(砍进度% 改情绪标签、24h 自动溶解、禁"逾期"字眼)已纳入功能红线。
- **制造新的情绪负担**:多一个不断闪烁的红点,连恋爱都要打卡完成 KPI。**缓解**:"提醒 vs 骚扰"的分界线是技术底座(F-02 场景感知分级 + F-12 去重摘要 + AI 主动降频而非加压催办)。负担的本质是"无差别提醒 + 单向催促",我们用 AI 把提醒变成"合适时机的、分级的、去重的",用红线把催促彻底移除——一个好的提醒系统应该让人感到"被托住"不是"被追赶"。
- **过度设计 / 高频琐事被 AI 过度包装**:买牛奶、缴物业费也生成五页方案会烦(老夫老妻共识)。**缓解**:AI 僚机限定在真正复杂、需要判断力的场景(报修档期协调、大宗采购比价、消耗品库存预测),提供"一句话→提醒→划掉"的零仪式极简通道;仪式感分级投放,不是每件事都值得"僚机觐见"。

## 13.2 社会舆论 / 性别风险

- **性别刻板 / 双向被质疑**:女王模式营销"双向"但首屏 / 推送 / 新手引导会本能预设一主一从,要么被女权 KOL 骂"强化刻板",要么被男性社群骂"凭什么她是陛下";"小狗"在豆瓣 #puppy play 讨论里已是被高度警惕的驯化符号;拉勾模式可能被读成"舔狗神器"、"她需要一个 App 才能让他做家务"被剪进吐槽视频(负向病毒传播)。**缓解**:五模式一律做角色 A ⇄ B 对称 UI + 性别中立文案审核;拉勾模式不做 AI 代写情话改"共同记忆小档案";含驯化 / 物化标签直接限流;提前写好三种主从争议下的舆情回应剧本。
- **韩国性别断层线(生死问题)**:见 §7.3——셔틀앱风险 + GS25 判例形状,必须约定 / 共创对称设计 + 亲族屏蔽 + 零分享,韩文文案本地人写 + 过"这一屏会怎么被截图"红队测试(非可选项)。

## 13.3 隐私 / 合规风险

- **亲密数据取证化(分手核弹)**:分手后"未办"工单 / 语音被截图、剪辑成"他根本不爱我"的证据,涉家暴 / 出轨 / 财产纠纷时带时间戳的语音 + 任务卡是现成电子证据。**缓解**:关系状态一键切"已分手"、双方独立确认后 72h 双向抹除原始数据(只留极简摘要);原始语音边录边转写即焚(服务器只存文本不存音频);法务明示"不构成法律证据";单方只能导出自己那份。
- **跨境数据 + 声纹刑事**:PIPC / APPI / BIPA 三国对"中国接收方跨境"和"声纹识别"审查极严(阿里 / Temu 已被罚,Otter.ai / Teams 因声纹被诉,韩国声纹=5 年以下有期徒刑)。**缓解**:数据不落中国 + 跨境同意写明目的地国名;第一版起就不做声纹 / 说话人识别(1v1 结对本就不需要,是免费合规红利);优先端侧 ASR;与 AI 供应商签 DPA。
- **资安敏感者的根本质疑**:长期开麦克风把对话 / 语气 / 财务 / 家庭矛盾传云端解析成"任务 + 情绪标签"=情侣关系的资料金矿。**缓解**:转写后即刻删除音频只留文本 + 明确 retention policy + 首次语音前电子书面同意 + 隐私营养标签如实填。

## 13.4 技术风险

- **免注册→账号迁移丢数据(生死级)**:现有用户全是匿名 token,迁移做砸=踩韩国"数据永不丢失"生死线。**缓解**:account 叠加不替换、旧 token 永久有效、迁移前后本地副本双保险、灰度上线、全程可回滚。
- **Cloudflare 单点 + D1 10GB 硬顶**:全栈押 Cloudflare,D1 单库上限 10GB 不可提额,`research_html` 大字段是隐患。**缓解**:v1 就把方案页 HTML 移 R2,监控 D1 用量,预研分库 / 迁 Postgres 路径。
- **Whisper 中文出繁体**:代码已注明,繁→简未做。**缓解**:runner / 后端加繁→简转换(opencc 类)或评估 SenseVoice(CJK 更强)。

## 13.5 竞争风险(平台夹击)

- **被大厂抄(最现实的商业威胁)**:微信 / iOS / 小米任何一次原生提醒能力升级或大模型入口内置"关系助手"都可能挤压空间;单点能力上我们没有护城河。**缓解(三条,都不是永久的)**:①关系数据的沉淀速度要快过巨头抄袭速度(数据复利,抄不走两个人沉淀两年的共同记忆);②大厂做不了"克制"——平台型产品基因是"增长 / 留存 / 时长",天然倾向做已读回执 / 在线状态 / 排行,而念念的整个价值建立在坚决不做这些之上,一个把"engagement 最大化"刻进 KPI 的大厂几乎不可能做出"故意让你说完就放下"的产品(护城河恰恰是我们敢于不做的那些事);③硬件 + 情绪信物是软件平台的射程盲区(需供应链和线下)。清醒地说三条护城河没有一条永久,所以商业策略写着"第一年是验证生意,不是收入生意",要在大厂反应过来之前验证"有人愿意为着落付钱"。
- **微信才是真正的敌人(而非同类 App)**:90% 情侣日常任务分配已在微信"语音+小作文+表情包"完成。**缓解**:做一件微信做不到的事——"在微信里,你帮对方做的事会消失在三天后的聊天记录里;在这里,它会变成一年后翻出来还能让你笑的纪念册。"差异化建在替代品做不到的三件事上:方向性协议(谁欠谁)、AI 从抱怨到方案、着落的仪式感(凭证 / 连胜 / 年度报告)。
- **冷启动双边问题(不回避的最大风险)**:必须两个人都装,而装 App 动机天然不对称(女方有痛、男方没有,男方视角这甚至是"一个专门给我派活的 App")。**缓解**:①单人可用先行(女方先单方记录 + AI 出方案,邀请男方是第二步);②把邀请本身产品化成一次示爱("我把想说的都放这了,你装了就不用怕忘");③给男方设计并度量 Aha 时刻(数字上不去先别营销)。

## 13.6 硬件风险

- **从浪漫道具滑向监控 / 家暴入口**:挂坠因"任务超时"在她那边震动=可穿戴的唠叨(全美国最容易导致离婚的产品);控制型伴侣把挂坠当单向情绪宣泄口、接收方不能屏蔽;老人误触导致子女开会中疯狂震动。**缓解**:接收方拥有绝对勿扰权(勿扰时段 / 白名单 / 震动强度 / 单日次数上限);「办妥」震动可从他那边发出(礼物),「你还没做」提醒绝不出现在她那边(审讯);7 天无双向互动挂坠自动进"思念休眠";硬件只走近场通讯、无 GPS / 无麦克风常开 / 按住才录;分手走"情感化退役"仪式流程。
- **供应链黑洞**:开模 + 认证 + 首批库存至少砸几百万,90% 硬件创业死在备货环节;三国都必须"整批进口 + 本地仓"(深圳直发对美韩走不通)。**缓解**:硬件当营销物料和获客引擎不当撑估值的业务(众筹是发布会不是生意,Lovebox / Filimin 品类天花板约"几十万台 + 年营收几百万美元");严格控制 SKU(一次只出一个挂坠);共板策略(挂链 / 钥匙扣 / 项链共用一块 PCBA);纽扣电池可绕开整条锂电池 KC 线(省 7–9 周,代价不可充电)。

## 13.7 风险优先级一句话总结

**产品与舆论风险靠"五条第一性原则 + 三红线"防守(这是宪法级);合规风险靠"不做声纹 + 数据不落中国 + 可导出可抹除"防守(有免费红利);竞争风险靠"沉淀速度 + 敢于不做 + 硬件盲区"防守(没有永久护城河,所以第一年是验证生意);冷启动与硬件风险是可被产品设计正面进攻的,不是结构性死局。**

## 结语:回到那条消息

这份主文档从头到尾,其实只在论证一件小到几乎微不足道、却又重到能压垮很多段感情的事:**让那些善意的、在场的、只是被生活的噪音冲散了的爱,不再漏接。** 我们不承诺让一个不爱的人变得爱你——那不是任何产品能做的事,承诺了就是骗子;我们只承诺让"我随口说的话,他都当真"这件事,被确认。

把这份报告压成一句话给三类读者:给老板,它是一份"读完就能开工"的总纲,十项待拍板清单在第 1.4 节;给团队,它是"哪个结论出自哪份调研"的可回溯地图,附录第 14 章是索引;给投资人,它是"为什么这件事今天才第一次能做、为什么别人卖甜我们卖作数、为什么第一年是验证生意"的完整论证。三类读者共用的那条主线始终没变——在这个即时性泛滥、他者正在消失、连接被广播稀释、亲密被秒回焦虑追赶的时代,我们想固执地做一件逆流的事:为两个人,重修一条最短的路。这条路上有距离,而我们珍视这段距离;这条路会穿越山海,而我们不让山海变成隔阂。她说的每一句,都有回响;他做的每一件,都被看见。

最后再钉一次这份报告最核心的判断,便于决策时随手回看:第一,及格线今天才第一次被够到(Avocado 之死是干净的反事实实验),这是时间窗口;第二,别人卖甜我们卖作数(付费假设钉死在"着落"上),这是定位命门;第三,五条第一性原则(去 KPI / 反监控 / 安全阀 / AI 只做成本 / 数据不丢)是全部克制的来源,也是对抗大厂的护城河(大厂做不了"克制");第四,先打念想 + 拉勾基建、用修勾破圈、首发日本、拉勾图形做全球符号——节奏比火力更重要;第五,第一年是验证生意不是收入生意,男方 Aha 达标率是那个"上不去就先别花一分钱营销"的总闸门。这五条立住了,才有资格谈规模化与融资。

念念不忘,必有回响。事事有回应,句句有着落。这就是为什么,这件事,值得做。
